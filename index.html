01
데이터 벡터화
Data Vectorization / Feature Engineering
MML §8.1

현실 데이터(텍스트, 이미지, 표 등)는 바로 수식에 넣을 수 없다. 기계학습 알고리즘은 수치 연산을 하기 때문에, 모든 입력을 반드시 실수 벡터 xᵢ ∈ ℝᴰ로 변환해야 한다. 이 변환 과정이 feature engineering이며, 결과로 얻어지는 각 차원의 값을 feature, attribute, covariate라고 부른다.

🔵 직관적 이해
사람의 정보를 "주민등록증 숫자들"로 정리하는 것과 같다. 이름, 성별 같은 텍스트 정보를 숫자 코드로 바꿔야 컴퓨터가 계산에 쓸 수 있다. 이 과정에서 어떤 숫자를 어떻게 부여하느냐가 모델 성능에 직접 영향을 미친다.

전처리 예시 (강의 예제 기준)

  • 이름(Name)예측에 무관한 정보 → 삭제 (dimensionality 감소)
  • 성별(Gender)이진 범주형 → 수치 레이블 (M = −1, F = +1) 또는 One-hot
  • 학위(Degree)순서형(ordinal) → 정수 레벨링 (BSc=1, MSc=2, PhD=3)
  • 우편번호(Postcode)지역 코드 → 위도/경도 실수로 변환 (지역 간 거리 반영)
  • 나이, 연봉이미 실수 → 정규화(standardization) 후 사용

범주형 변수 인코딩 방법 비교

방법방식적합한 경우단점
Label Encoding각 범주에 정수 부여 (0, 1, 2 ...)순서가 있는 범주 (학위 등)순서 없는 범주에 적용 시 크기 관계가 생김
One-hot EncodingK개 범주 → K차원 이진 벡터순서 없는 명목형 범주차원(D)이 크게 증가할 수 있음
핵심 원칙
원시 데이터 → 전처리 → 실수 벡터 xᵢ ∈ ℝᴰ → ML 알고리즘 투입.
정규화(Normalization): 서로 다른 스케일의 feature들을 동등하게 다루기 위해 평균 0, 분산 1로 표준화하는 경우가 많다 (Standardization: x' = (x − μ)/σ).
02
모델의 종류
Function Model vs. Probabilistic Model
MML §8.1

같은 입력 데이터에 대해 출력을 어떻게 정의하느냐에 따라 두 가지 모델이 존재한다. 선택한 모델 종류에 따라 사용할 수 있는 학습 방법론이 달라진다.

🔵 직관적 이해
함수 모델: "60세 직원의 연봉은 8,000만원이다" — 단 하나의 확정적 값 반환.
확률 모델: "60세 직원의 연봉은 N(8000, 1500²)만원 분포를 따른다" — 불확실성까지 표현.
현실은 항상 불확실하므로, 확률 모델이 더 풍부한 정보를 제공한다. 다만 계산 비용이 더 높다.
구분
함수 모델 (Function Model)
확률 분포 모델 (Probabilistic Model)
출력 형태
하나의 결정론적 값 ŷ ∈ ℝ
확률 분포 p(y | x)
표현 방식
f : ℝᴰ → ℝ, ŷ = f(x, θ)
p(y | x, θ) — 조건부 확률 분포
불확실성 표현
불가능 (점 예측만 가능)
가능 (예측 분포의 분산으로 표현)
학습 방법
ERM (경험적 위험 최소화)
MLE / MAP / Bayesian Inference
오버피팅 제어
정규화(Regularization) 항 추가
사전 분포(Prior) p(θ) 부여
핵심 연결
두 모델은 따로 동작하는 것이 아니다. 확률 모델에서 Gaussian 우도를 가정하고 MLE를 수행하면 수학적으로 함수 모델의 ERM(최소자승)과 같아진다. 또한 Gaussian prior를 가정하고 MAP를 수행하면 L2 정규화 ERM과 같아진다. (→ 개념 7, 8에서 상세히 다룸)
03
학습의 구성
ML Pipeline: Learning = Parameter Finding
MML §8.2

기계학습 알고리즘은 세 단계로 구성된다. 이 세 단계는 서로 다른 데이터 셋에서, 서로 다른 목적으로 수행된다.

1
예측/추론 (Prediction / Inference): 이미 학습된 모델에 새로운 입력 x를 넣어 ŷ 또는 p(y|x)를 계산한다. 이 단계에서는 파라미터를 변경하지 않는다. → Test set 사용
2
파라미터 추정 (Training / Parameter Estimation): 주어진 훈련 데이터로 손실 함수를 최소화(ERM) 또는 우도를 최대화(MLE/MAP)하여 최적 θ*를 찾는다. → Training set 사용
3
모델 선택 (Hyperparameter Tuning / Model Selection): 함수의 형태, 모델 복잡도(다항식 차수 d), 정규화 강도 λ 등 하이퍼파라미터를 결정한다. → Validation set 또는 Cross-Validation 사용
파라미터 vs. 하이퍼파라미터
파라미터(θ): 데이터로부터 학습되는 값. 예: 선형회귀의 기울기 a, 절편 b, 신경망의 가중치 W.
하이퍼파라미터: 사람이 미리 정하는 값. 예: 정규화 강도 λ, 다항식 차수 d, 신경망 레이어 수. 데이터로 학습되지 않으며, 교차검증(Cross-Validation)으로 최적값 탐색.
⚠️ 시험 포인트
하이퍼파라미터를 Test set으로 최적화하면 information leakage가 발생 — test set이 사실상 training에 사용된 셈이 되어 일반화 성능 추정이 불가능해진다. 반드시 별도의 Validation set 또는 CV를 사용해야 한다.
04
경험적 위험 최소화
Empirical Risk Minimization (ERM)
MML §8.2

예측인자(predictor)가 함수형일 때, 훈련 데이터에서의 평균 오차(경험적 위험)를 최소화하여 파라미터를 찾는 방법이다.

🔵 왜 "경험적"인가?
우리가 진짜 원하는 것은 미래의 모든 데이터에 대해 잘 동작하는 함수(Expected Risk 최소화)이다. 하지만 미래 데이터는 볼 수 없으므로, 지금 가진 N개의 훈련 데이터(경험)로 이를 근사한다. 이것이 바로 경험적 위험(Empirical Risk)이다.
Loss Function — 예측 오차 측정
ℓ(yₙ, ŷₙ)실제값 yₙ과 예측값 ŷₙ의 차이를 non-negative 값으로 변환. 항상 ≥ 0
Empirical Risk (경험적 위험)
R_emp(f, X, y) = (1/N) Σₙ ℓ(yₙ, ŷₙ)N개 훈련 샘플의 평균 손실. ŷₙ = f(xₙ, θ)
Expected Risk (기대 위험) — 진짜 목표
R_true(f) = E_{x,y}[ℓ(y, f(x))]데이터 분포 전체에 대한 기대 손실 → 미래 데이터에서의 실제 성능
Least-Squares Loss 적용 (예제 8.2) — 선형 예측자 ŷₙ = θᵀxₙ
ℓ(yₙ, ŷₙ) = (yₙ − ŷₙ)²제곱 오차 손실 함수
min_θ (1/N) Σₙ (yₙ − θᵀxₙ)²= min_θ (1/N) ‖y − Xθ‖²
해: θ* = (XᵀX)⁻¹ Xᵀy정규 방정식(Normal Equation) — closed-form 해

i.i.d. 가정 (Independent and Identically Distributed): 훈련 데이터 (x₁,y₁), ..., (xₙ,yₙ)가 서로 독립이고 같은 분포에서 추출되었다고 가정한다. 이 가정 덕분에 결합 확률이 곱의 형태로 분해되어 계산이 가능해진다.

i.i.d. 가정이 주는 수학적 이점
p(y₁,...,yₙ | x₁,...,xₙ, θ) = ∏ₙ p(yₙ | xₙ, θ)독립 → 곱으로 분해 → log 취하면 합으로 변환
⚠️ ERM의 핵심 한계
R_emp가 작다고 R_true도 작은 것은 아니다. 모델이 복잡할수록 R_emp ↓, R_true ↑ 의 오버피팅(overfitting)이 발생한다. 이것이 정규화와 교차검증이 필요한 근본 이유다.
05
오버피팅 & 정규화
Overfitting & Regularization
MML §8.2.3

오버피팅(Overfitting): 모델이 훈련 데이터의 잡음(noise)까지 학습하여, 훈련 오차(R_emp)는 매우 낮지만 새로운 데이터에서의 기대 오차(R_true)는 크게 높은 현상. 훈련 데이터에만 "특화"된 복잡한 함수가 만들어진다.

세 가지 피팅 상태 비교
Underfitting: 모델이 너무 단순 → R_emp 높음, R_true 높음. (High Bias)
Good Fitting: 적절한 복잡도 → R_emp 적당, R_true ≈ R_emp. 일반화 성공.
Overfitting: 모델이 너무 복잡 → R_emp 매우 낮음, R_true ≫ R_emp. (High Variance)
🔵 Bias-Variance Trade-off
오버피팅과 언더피팅의 관계는 Bias-Variance Trade-off로 설명된다:
Bias(편향): 모델이 진짜 함수에서 체계적으로 벗어나는 정도. 너무 단순한 모델 → High Bias.
Variance(분산): 다른 훈련 데이터에 따라 모델이 얼마나 달라지는지. 너무 복잡한 모델 → High Variance.
λ(정규화 강도) 증가 → Bias↑, Variance↓ / λ 감소 → Bias↓, Variance↑

해결책: 정규화(Regularization) — 손실 함수에 파라미터 크기에 대한 페널티 항을 추가하여 모델 복잡도를 제어한다.

Regularized Least-Squares (예제 8.3) — Ridge Regression
기본: min_θ (1/N) ‖y − Xθ‖²
정규화: min_θ (1/N) ‖y − Xθ‖² + λ‖θ‖²λ: 정규화 파라미터 (하이퍼파라미터)
최적해: θ* = (XᵀX + NλI)⁻¹ Xᵀyλ>0이면 XᵀX가 singular여도 역행렬 존재 → 수치 안정성
  • λ = 0정규화 없음. 일반 최소자승 해와 동일. 오버피팅 위험 있음.
  • λ → ∞페널티가 압도적 → θ* → 0. 극도로 단순한 모델 (underfitting).
  • 최적 λCross-Validation으로 결정. Bias↑ Variance↓ trade-off 균형점 탐색.
⚠️ 정규화 ≠ 사전 분포(Prior)
수식은 같지만 해석이 다르다: ERM 관점에서는 "페널티 항"이지만, MAP 관점에서는 "Gaussian prior의 음의 로그 확률"이다. λ‖θ‖² ↔ −log N(θ | 0, I/λ). (→ 개념 08에서 자세히)
06
교차검증
K-fold Cross-Validation
MML §8.2.4

미래 데이터에서의 성능(Expected Risk)을 추정하기 위해, 보유한 데이터를 K개의 fold(chunk)로 나눠 K−1개를 훈련에, 1개를 검증에 사용하는 방법. 이를 K번 반복하여 평균을 낸다.

🔵 왜 CV가 필요한가?
단순 train/test split은 결과가 어떻게 나누었느냐에 따라 크게 달라진다 (높은 분산). K-fold CV는 모든 데이터를 한 번씩 validation에 사용하므로, 추정치의 분산이 낮고 데이터를 효율적으로 활용할 수 있다.
1
전체 훈련 데이터를 K개의 동등한 크기의 fold로 분할: D = D⁽¹⁾ ∪ D⁽²⁾ ∪ ... ∪ D⁽ᴷ⁾
2
k = 1, ..., K에 대해: k번째 fold를 validation set V⁽ᵏ⁾으로, 나머지 K−1개를 training set R⁽ᵏ⁾으로 사용하여 모델 f⁽ᵏ⁾ 학습
3
각 반복에서 validation risk R(f⁽ᵏ⁾, V⁽ᵏ⁾) 계산 → K번 평균
K-fold Generalization Error 추정
E_V[R(f, V)] ≈ (1/K) Σₖ R(f⁽ᵏ⁾, V⁽ᵏ⁾)
조건: R⁽ᵏ⁾ ∪ V⁽ᵏ⁾ = D, R⁽ᵏ⁾ ∩ V⁽ᵏ⁾ = ∅ (모든 데이터 활용, 겹침 없음)
K 값특성장점단점
K = 5 또는 10일반적으로 사용계산 비용과 추정 품질의 균형—
K = N (LOOCV)Leave-One-Out CV편향이 가장 낮음, 데이터 최대 활용계산 비용이 매우 큼 (N번 학습)
K = 250:50 split빠른 계산추정 분산 큼
Nested CV (결합 교차검증) — 모델 선택용
Inner loop: 여러 후보 하이퍼파라미터(λ₁, λ₂, ...) 각각에 대해 K-fold CV 수행 → generalization error 추정 → 가장 낮은 error의 하이퍼파라미터 선택.
Outer loop: 전체 training data로 선정된 최적 하이퍼파라미터로 재학습 → 별도로 떼어 둔 test set으로 최종 성능 평가 (한 번만 사용).

이중 구조인 이유: inner loop에서 쓴 validation data로는 성능을 정직하게 평가할 수 없다. 이미 모델 선택에 사용되었기 때문이다.
07
최대 우도 추정
Maximum Likelihood Estimation (MLE)
MML §8.3

관측된 데이터를 가장 잘 설명하는 파라미터 θ를 찾는 방법. "어떤 θ 값이었어야 이 데이터가 나올 확률이 제일 높은가?"를 묻는다.

🔵 직관: 동전 던지기 비유
동전을 10번 던져 7번 앞면이 나왔다. 앞면 확률 θ에 대한 MLE는? → "이 결과(7번 앞면)가 나올 확률을 최대로 만드는 θ = 7/10 = 0.7"이다. 즉, 데이터가 가장 그럴듯하게(likely) 보이는 파라미터를 선택한다.
1
우도 함수 작성: i.i.d. 가정 → 결합 우도 = 각 샘플 우도의 곱
p(Y | X, θ) = ∏ₙ₌₁ᴺ p(yₙ | xₙ, θ)
2
로그 변환: 곱 → 합으로 변환하여 계산 안정화 및 미분 편의
log p(Y | X, θ) = Σₙ log p(yₙ | xₙ, θ)
3
부호 반전 후 최소화: 최대화 → 음수 취해 최소화 문제로 전환 (optimizer는 보통 최소화)
L(θ) = −Σₙ log p(yₙ | xₙ, θ) → min
4
최적화: ∂L/∂θ = 0 풀기 또는 Gradient Descent
Gaussian 우도 → 최소자승 문제 (예제 8.5) — 핵심 유도
p(yₙ | xₙ, θ) = N(yₙ | xₙᵀθ, σ²) 가정
L(θ) = −Σₙ log N(yₙ | xₙᵀθ, σ²)
= (1/2σ²) Σₙ(yₙ−xₙᵀθ)² + N·log√(2πσ²)
σ가 상수이면 두 번째 항은 θ와 무관 → argmin_θ L(θ) = argmin_θ Σₙ(yₙ−xₙᵀθ)² ← 최소자승!
MLE의 특성과 한계
Asymptotic Consistency: N → ∞이면 θ_MLE → θ_true (실제 파라미터로 수렴).
Asymptotic Normality: N이 충분히 크면 θ_MLE는 Gaussian 분포를 따름.
Small data overfitting: 데이터가 적으면 훈련 데이터에 과도하게 fitting → overfitting 발생.
사전 지식 반영 불가: θ에 대한 사전 정보(prior)를 활용하지 않는다. → MAP로 해결.
⚠️ log를 취하는 두 가지 이유
① 수치 안정성: 확률값 p ∈ (0,1]을 N번 곱하면 float underflow 발생 (≈ 0으로 수렴). log를 취해 합으로 변환하면 안전하다.
② 최적화 용이: log는 순증가 함수이므로 argmax가 보존된다. 또한 log의 미분이 1/p로 단순하여 gradient 계산이 편리하다.

최대 사후 추정
Maximum A Posteriori (MAP)
MML §8.3

MLE에 사전 분포(prior) p(θ)를 추가하여 θ에 대한 사전 지식을 반영하는 방법. 데이터를 보기 전에 θ가 어떨 것이라는 믿음(prior)과, 데이터가 주는 정보(likelihood)를 결합하여 사후 확률(posterior)을 최대화한다.

🔵 직관: 의사 진단 비유
환자가 기침을 한다(데이터). 기침의 원인으로 감기(M₁)와 폐암(M₂) 중 어느 것이 더 그럴듯한가?
MLE: "기침 증상을 설명하는 확률만 비교" → 폐암도 기침을 설명 가능하므로 M₂를 선택할 수도 있다.
MAP: "기침 확률 × 사전에 해당 질병 가능성" → 폐암보다 감기의 사전 확률(prior)이 훨씬 높으므로 M₁(감기) 선택. 더 합리적!
Bayes' Theorem (베이즈 정리)
p(θ | X) = p(X | θ) p(θ) / p(X)
posterior ∝ likelihood × prior
p(X)는 θ에 무관한 정규화 상수 → 최적화에서 무시 가능
MAP Estimate (후위 확률 최대화)
θ_MAP = argmax_θ p(θ | X, Y)
= argmax_θ [log p(Y | X, θ) + log p(θ)]
= 음의 NLL + 음의 log prior 의 합을 최소화
Gaussian Prior → L2 정규화 동치 유도 (예제 8.6)
p(θ) = N(θ | 0, b²I) 가정
log p(θ) = −(1/2b²)‖θ‖² + const
MAP = argmin_θ [(1/N)‖y−Xθ‖² + (σ²/b²N)‖θ‖²]
λ = σ²/b² 로 놓으면: MAP ≡ Ridge Regression
∴ Gaussian prior MAP = L2 정규화 ERM. 정규화 파라미터 λ는 prior 분산 b²의 역수에 비례.
구분MLEMAP
목표p(Y|X,θ) 최대화p(θ|X,Y) ∝ p(Y|X,θ)·p(θ) 최대화
사전 지식없음 (uniform prior 가정과 동일)있음 — prior p(θ) 명시
추정 결과점 추정 θ*점 추정 θ_MAP
정규화 동치없음Gaussian prior → L2 / Laplace prior → L1
N → ∞데이터가 지배prior 영향 감소, MLE로 수렴

베이지안 추론
Bayesian Inference
MML §8.4

MLE/MAP가 θ의 한 점(point estimate)을 추정하는 것과 달리, 베이지안 추론은 파라미터 분포 p(θ|X) 전체를 추정한다. θ를 확률 변수로 취급하여 불확실성을 분포로 표현한다.

🔵 세 방법의 핵심 철학 차이
MLE: "가장 그럴듯한 θ 하나만 찾아 쓴다" → 불확실성 무시
MAP: "사전 지식도 반영한 가장 그럴듯한 θ 하나를 찾아 쓴다" → 불확실성 무시
Bayesian: "θ에 대한 모든 가능성을 확률 분포로 유지하고, 예측할 때 전부 평균낸다" → 파라미터 불확실성(model uncertainty) 정량화
Posterior Distribution (베이즈 추론의 핵심)
p(θ | X) = p(X | θ) p(θ) / p(X)
p(X) = ∫ p(X | θ) p(θ) dθ← marginal likelihood (evidence). 정규화 상수. 계산이 매우 어렵다.
Predictive Distribution (새로운 데이터 예측)
p(x_new | X) = ∫ p(x_new | θ) p(θ | X) dθ
= E_{θ|X}[p(x_new | θ)]posterior로 가중된 모든 θ에 대한 예측 분포의 평균
MLE/MAP 대비 Bayesian 예측의 차이
MLE/MAP: θ* 하나를 고정 → p(x_new | θ*)만 사용. 파라미터 추정 불확실성 무시.
Bayesian: 가능한 모든 θ를 확률적으로 평균 → 파라미터 불확실성을 예측에 반영.
결과: Bayesian 예측 분포는 MLE/MAP보다 더 넓은 분산을 가진다. 특히 훈련 데이터가 적은 영역에서 불확실성을 올바르게 표현한다.
⚠️ 베이지안 추론의 계산 문제
p(X) = ∫ p(X|θ) p(θ) dθ 적분이 대부분 closed-form 불가.
해결책 1 — Conjugate Prior: likelihood와 prior가 "켤레(conjugate)" 관계이면 posterior도 같은 분포 족 → closed-form 계산 가능. 예: Gaussian likelihood + Gaussian prior → Gaussian posterior.
해결책 2 — 수치 근사: Monte Carlo (MCMC), Variational Inference 등.
방법추정 결과사전 지식계산 방법계산 복잡도
MLE점 추정 θ*없음최적화 (∇L=0)낮음
MAP점 추정 θ_MAPprior p(θ)최적화 (∇(L+logp)=0)낮음~중간
Bayesian분포 p(θ|X)prior p(θ)적분 (전체 posterior)매우 높음

잠재 변인 모델
Latent Variable Model
MML §8.4

관측 데이터 x 외에 숨겨진 변인(latent variable) z가 존재한다고 가정하여, 모델의 표현력을 높이는 방법. z는 직접 관측되지 않지만 데이터 생성 과정에 관여한다고 가정한다.

🔵 직관: 혼합 모델(Mixture Model)
전국 성인 키 데이터를 가지고 있다. 이 데이터는 사실 남성과 여성의 두 Gaussian이 섞여 있다.
z = 각 데이터가 남성(z=0)인지 여성(z=1)인지 → 관측 불가능한 잠재 변인.
z를 모르더라도, z의 prior p(z)와 조건부 분포 p(x|z)를 가정하면 전체 분포를 모델링 가능. 이것이 Gaussian Mixture Model(GMM)이다.
z를 Marginalize하여 관측 데이터 우도 계산
p(x | θ) = ∫ p(x | z, θ) p(z) dzz의 prior p(z)로 가중하여 평균 → z가 제거됨
z의 사후 확률 (Posterior of z)
p(z | x, θ) = p(x | z, θ) p(z) / p(x | θ)"이 데이터 x가 어떤 성분(z)에서 나왔을 확률"
Bayesian 관점 — θ의 Posterior
p(θ | X) = p(X | θ) p(θ) / p(X)
p(X | θ) = ∫ p(X | z, θ) p(z) dzz를 marginalize한 결과를 likelihood로 사용
켤레 사전 분포(Conjugate Prior)의 중요성
p(z)가 p(x|z, θ)의 켤레(conjugate)이면 → posterior p(z|x,θ)가 p(z)와 같은 분포 족 → closed-form 계산 가능.
켤레가 아니면 → EM 알고리즘, MCMC, Variational Inference 등 수치 근사 방법 필요.
주요 켤레 쌍: Gaussian likelihood ↔ Gaussian prior, Bernoulli likelihood ↔ Beta prior, Multinomial ↔ Dirichlet
활용 예
MML 10장 — PCA (z: 저차원 표현), 11장 — GMM (z: 군집 소속), Hidden Markov Model (z: 숨겨진 상태), VAE (z: 잠재 공간 벡터)
11
모델 선택 & 베이즈 요인
Model Selection & Bayes Factor
MML §8.6

오캄의 면도날(Occam's Razor): "데이터를 설명하기에 충분한 가장 단순한 모델이 최선이다." 복잡한 모델은 overfitting 위험이 높고, 단순한 모델은 underfitting 위험이 있다. 최적의 복잡도를 어떻게 자동으로 선택하는가?

베이지안 모델 선택
p(Mₖ | D) ∝ p(Mₖ) · p(D | Mₖ)
p(D | Mₖ) = ∫ p(D | θₖ) p(θₖ | Mₖ) dθₖ← Marginal Likelihood (모델 증거, evidence)
M* = argmax_{Mₖ} p(Mₖ | D)← 가장 높은 posterior 확률의 모델 선택
Bayes Factor (모델 비교 지표)
Posterior Odds = Prior Odds × Bayes Factor
p(M₁|D)/p(M₂|D) = [p(M₁)/p(M₂)] × [p(D|M₁)/p(D|M₂)]
Bayes Factor = p(D|M₁)/p(D|M₂) — 데이터 관점에서의 두 모델 지지도 비율
  • BF > 1데이터가 M₁을 더 지지. BF > 3: moderate, BF > 10: strong evidence
  • Prior Odds = 1p(M₁) = p(M₂) 이면 → Posterior Odds = Bayes Factor
  • Uniform prior모든 모델이 동등하게 그럴듯 → Marginal Likelihood만으로 비교
🔵 Marginal Likelihood가 오캄의 면도날을 자동 구현하는 이유
p(D | Mₖ) = ∫ p(D | θₖ) p(θₖ | Mₖ) dθₖ

복잡한 모델: 파라미터 공간이 넓음 → prior p(θ|M)이 넓게 퍼짐 → 각 θ에서의 밀도가 낮음 → 확률이 "희석"됨 → marginal likelihood 작아짐.
단순한 모델: 파라미터 공간이 좁음 → prior가 집중됨 → 데이터와 잘 맞는 θ 부근에서 밀도가 높음 → marginal likelihood 클 수 있음.

결론: marginal likelihood는 데이터 설명력(likelihood)과 모델 복잡도(prior의 퍼짐도) 간의 자동 trade-off를 구현한다. 불필요하게 복잡한 모델은 자연스럽게 불이익을 받는다.
⚠️ 모델 선택에서 주의할 점
Marginal Likelihood 계산 자체도 어렵다 (적분 필요). 실제로는 BIC(Bayesian Information Criterion), AIC 등의 근사 지표를 사용하는 경우가 많다.
또한 모델 선택 시 test set을 한 번도 사용하지 말아야 한다 — Nested CV에서 outer loop test set은 최종 성능 평가에만 쓰인다.

풀이

J(θ) = (1/N)‖y − Xθ‖² + λ‖θ‖²

∇_θ J = (2/N)Xᵀ(Xθ − y) + 2λθ = 0

(1/N)XᵀXθ + λθ = (1/N)Xᵀy

Q5 MLE에서 i.i.d. 가정 하에 결합 우도를 Negative Log-Likelihood(NLL)로 변환하는 과정을 단계별로 유도하고, log를 취하는 두 가지 이유(수치적, 최적화적)를 설명하라. 유도 +
풀이
Step 1 — i.i.d. 가정으로 결합 우도 분해:
p(Y | X, θ) = ∏ₙ₌₁ᴺ p(yₙ | xₙ, θ)
Step 2 — log 변환 (단조증가 → argmax 보존):
log p(Y | X, θ) = Σₙ₌₁ᴺ log p(yₙ | xₙ, θ)
Step 3 — 부호 반전하여 최소화 문제로 변환:
L(θ) = −log p(Y | X, θ) = −Σₙ log p(yₙ | xₙ, θ)
θ_MLE = argmin_θ L(θ)
log를 취하는 두 가지 이유:
① 수치 안정성: p ∈ (0,1]을 N번 곱하면 float underflow 발생 (≈ 0). log를 취하면 합으로 변환되어 underflow 방지.
② 최적화 편의: log는 순증가 함수 → argmax 동일. 또한 ∂log p/∂θ = (1/p)·∂p/∂θ로 연쇄 미분이 단순해짐.
Q6 Gaussian 우도 p(yₙ|xₙ,θ) = N(yₙ | xₙᵀθ, σ²) 하에서 MLE를 수행하면 최소자승 문제와 수학적으로 동일함을 완전히 유도하라. (MML 예제 8.5) 유도 +
풀이
Gaussian 분포의 정의를 NLL에 대입:
L(θ) = −Σₙ log N(yₙ | xₙᵀθ, σ²)
Gaussian의 pdf: N(y|μ,σ²) = (2πσ²)^{−1/2} exp(−(y−μ)²/2σ²)
= −Σₙ [−(yₙ−xₙᵀθ)²/(2σ²) − (1/2)log(2πσ²)]
= (1/2σ²) Σₙ(yₙ−xₙᵀθ)² + (N/2)log(2πσ²)
σ²가 주어진 상수이므로, θ에 관한 최소화에서 두 번째 항은 상수:
argmin_θ L(θ) = argmin_θ (1/2σ²) Σₙ(yₙ−xₙᵀθ)²
= argmin_θ Σₙ(yₙ−xₙᵀθ)² ← 최소자승 문제
∴ Gaussian 우도 하의 MLE ≡ 최소자승 문제. 두 방법은 수학적으로 동일한 최적해를 갖는다.
Q7 MAP 추정에서 파라미터 θ에 Gaussian prior p(θ) = N(θ | 0, b²I)를 부여하면, 왜 Ridge Regression(L2 정규화 최소자승)과 동일한 결과가 나오는지 유도하라. 정규화 파라미터 λ가 b²와 어떤 관계인지 도출하라. 유도 +
풀이
MAP: θ_MAP = argmax_θ [log p(Y|X,θ) + log p(θ)]

Step 1 — Gaussian prior의 log:
log p(θ) = log N(θ | 0, b²I) = −(1/2b²)θᵀθ − const = −‖θ‖²/(2b²) − const
Step 2 — Gaussian 우도(→ 최소자승)와 결합:
MAP = argmax_θ [−(1/2σ²)‖y−Xθ‖² − (1/2b²)‖θ‖²]
Step 3 — 부호 반전하여 최소화:
= argmin_θ [(1/2σ²)‖y−Xθ‖² + (1/2b²)‖θ‖²]
(1/N) 정규화하면:
= argmin_θ [(1/N)‖y−Xθ‖² + (Nσ²/Nb²)‖θ‖²] ≡ argmin_θ [(1/N)‖y−Xθ‖² + λ‖θ‖²]
∴ λ = σ²/b²
정규화 파라미터 λ는 노이즈 분산 σ²에 비례하고, prior 분산 b²에 반비례한다.
b²가 작을수록(prior가 확신) λ가 커지고, prior의 영향이 강해진다.
Q8 다음 조건에서 MLE와 MAP 추정값을 각각 구하라. 데이터: y₁=1, y₂=3, y₃=5 (xₙ=n, N=3), 모델: yₙ = θxₙ + εₙ, ε~N(0,1). MAP의 경우 prior: θ~N(0, 1) 사용. λ = σ²/b² = 1/1 = 1. 계산 +
풀이
데이터: x = [1,2,3]ᵀ, y = [1,3,5]ᵀ

MLE (정규 방정식):
XᵀX = 1²+2²+3² = 1+4+9 = 14
Xᵀy = 1·1+2·3+3·5 = 1+6+15 = 22
θ_MLE = (XᵀX)⁻¹Xᵀy = 22/14 = 11/7 ≈ 1.571
MAP (Ridge with λ=1, N=3):
θ_MAP = (XᵀX + NλI)⁻¹Xᵀy = (14 + 3×1)⁻¹ × 22 = 22/17 ≈ 1.294
해석: MAP 값(1.294)이 MLE(1.571)보다 0 방향으로 수축(shrinkage)되었다. prior N(0,1)이 θ를 원점 쪽으로 당기는 정규화 효과를 발휘한 결과.
Q9 MLE, MAP, Bayesian Inference를 (1) 추정 결과 형태, (2) 사전 지식 활용, (3) 계산 방법, (4) 오버피팅 저항성, (5) 데이터 N이 매우 클 때의 수렴 방향의 다섯 가지 측면에서 비교하라. 비교 +
풀이
항목 | MLE | MAP | Bayesian -----------------|------------------|----------------------|------------------ 추정 결과 | 점 추정 θ* | 점 추정 θ_MAP | 분포 p(θ|X) 사전 지식 | 없음 (uniform) | prior p(θ) 반영 | prior p(θ) 반영 계산 방법 | 최적화 (∇L=0) | 최적화 (prior 포함) | 적분 (전체 posterior) 계산 비용 | 낮음 | 낮음~중간 | 매우 높음 오버피팅 저항 | 약함 | 중간 (prior 강도 의존)| 강함 (분포 유지) N→∞ 수렴 | θ_true 수렴 | θ_MLE와 수렴 | 점 mass at θ_true
실용적 선택 기준:
데이터 많음 → MLE / 사전 지식 있거나 데이터 적음 → MAP / 불확실성 정량화 필요 → Bayesian
Q10 베이지안 추론에서 predictive distribution p(x_new | X)를 구하는 과정을 수식으로 서술하라. 이것이 MLE 방식의 p(x_new | θ*) 사용과 근본적으로 다른 점을 두 가지 이상 설명하라. 유도 +
풀이
Bayesian Predictive Distribution:
p(x_new | X) = ∫ p(x_new | θ) p(θ | X) dθ = E_{p(θ|X)}[p(x_new | θ)]
과정: ① posterior p(θ|X) 계산 (학습 단계) ② x_new에 대해 가능한 모든 θ로 예측한 분포를 posterior로 가중 평균.

MLE 방식과의 차이:
① 파라미터 불확실성 반영: MLE는 θ* 하나로 예측. 베이지안은 posterior에 따라 모든 가능한 θ를 평균 → 파라미터 추정 오차가 예측 분포의 분산에 반영됨.
② 예측 분포의 분산: 베이지안 예측 분포는 항상 MLE 예측 분포보다 넓거나 같은 분산을 가진다. 데이터가 적은 영역에서 불확실성이 자동으로 증가.
③ θ에 종속되지 않음: MLE 예측은 추정된 θ*에 전적으로 의존하지만, 베이지안 예측은 θ를 marginalize하여 파라미터 없이 데이터만의 함수로 표현됨.
Q11 두 모델 M₁, M₂에 대해 p(M₁)=0.3, p(M₂)=0.7, p(D|M₁)=0.5, p(D|M₂)=0.2가 주어졌다. (1) Bayes Factor를 계산하라. (2) Prior Odds를 계산하라. (3) Posterior Odds를 계산하여 최종적으로 어떤 모델을 선택해야 하는지 판단하라. 계산 +
풀이
(1) Bayes Factor = p(D|M₁) / p(D|M₂) = 0.5 / 0.2 = 2.5
(2) Prior Odds = p(M₁) / p(M₂) = 0.3 / 0.7 ≈ 0.4286
(3) Posterior Odds = Prior Odds × Bayes Factor = 0.4286 × 2.5 ≈ 1.071
Posterior Odds > 1이므로 M₁ 선택.

해석: 사전 확률(Prior Odds = 0.43)은 M₂를 선호했지만, 데이터 설명력(Bayes Factor = 2.5)은 M₁을 강하게 지지한다. 두 효과를 결합한 결과(Posterior Odds ≈ 1.07)는 M₁이 약간 우세하다. 증거가 강하지 않으므로 추가 데이터 수집이 권장될 수 있다.
Q12 오캄의 면도날(Occam's Razor) 원칙이 베이지안 모델 선택에서 marginal likelihood를 통해 자동으로 구현되는 이유를 설명하라. 복잡한 모델과 단순한 모델의 marginal likelihood 크기가 어떻게 차이 나는지 prior 분포의 관점에서 논하라. 비교 +
풀이
오캄의 면도날: "데이터를 설명하기에 충분한 가장 단순한 모델이 최선"
p(D | Mₖ) = ∫ p(D | θₖ) p(θₖ | Mₖ) dθₖ
복잡한 모델:
파라미터 공간이 넓음 → prior p(θ|M)이 넓게 퍼져 있음 → 각 θ 지점에서의 prior 밀도가 낮음 → 비록 likelihood p(D|θ)가 최적점에서 높더라도 prior 가중치가 낮아 적분 값이 작아짐 (확률 희석 효과).

단순한 모델:
파라미터 공간이 좁음 → prior가 집중됨 → 데이터를 잘 설명하는 θ 근방에서 prior 밀도가 높음 → 적분 결과 즉 marginal likelihood가 클 수 있음.

결론: marginal likelihood는 데이터 설명력(likelihood)과 모델 복잡도(prior의 퍼짐도) 간의 자동 trade-off를 반영한다. 불필요하게 복잡한 모델은 확률 희석으로 인해 자연스럽게 불이익을 받는다. 이것이 오캄의 면도날의 베이지안 구현이다.
Q13 켤레 사전 분포(Conjugate Prior)의 정의를 서술하고, Gaussian likelihood와 켤레를 이루는 prior는 무엇인지 밝혀라. 켤레 prior가 중요한 수학적 이유와 켤레가 아닐 때의 대안적 계산 방법을 설명하라. 개념 +
풀이
켤레 사전 분포의 정의:
likelihood p(x|θ)와 prior p(θ)가 켤레(conjugate) 관계이면, posterior p(θ|x)가 prior와 같은 분포 족(family)에 속한다.
p(θ|x) ∝ p(x|θ)·p(θ) → posterior ∈ 같은 분포 족
Gaussian likelihood의 켤레 prior:
p(x|θ) = N(x | θ, σ²) 에 대해 → prior p(θ) = N(θ | μ₀, σ₀²) → posterior p(θ|x) = N(θ | μ_n, σ_n²) (역시 Gaussian).

중요한 수학적 이유:
베이즈 정리의 분모 p(X) = ∫ p(X|θ)p(θ)dθ 적분이 closed-form으로 계산 가능해진다. 그렇지 않으면 이 적분은 대부분 해석적 해가 없다.

켤레가 아닐 때의 대안:
① MCMC (Markov Chain Monte Carlo): 샘플링으로 posterior 근사.
② Variational Inference: posterior를 단순한 분포로 근사하여 최적화.
Q14 잠재 변인 모델(Latent Variable Model)에서 잠재 변인 z를 marginalize하여 관측 데이터 우도 p(x|θ)를 얻는 과정을 수식으로 서술하라. 왜 z를 직접 추정하지 않고 marginalize하는가? GMM을 예시로 들어 설명하라. 개념 +
풀이
z의 Marginalization:
p(x | θ) = ∫ p(x | z, θ) p(z) dz (연속 z)
p(x | θ) = Σ_z p(x | z, θ) p(z) (이산 z)
왜 직접 추정하지 않고 marginalize하는가:
z는 관측 불가능한 변인이다. z를 고정하면 과제가 단순해지지만, z의 참값을 모르므로 불확실한 상태로 남겨야 한다. Marginalization은 z에 대한 불확실성(prior p(z))을 반영한 채로 우도를 계산하는 올바른 방법이다.

GMM 예시:
데이터 xₙ이 K개 Gaussian 성분 중 하나에서 나왔다고 가정. z_n ∈ {1,...,K} (관측 불가):
p(xₙ | θ) = Σₖ₌₁ᴷ p(xₙ | z_n=k, θ) p(z_n=k) = Σₖ πₖ N(xₙ | μₖ, Σₖ)
πₖ = p(z=k): 혼합 계수(mixing coefficient). z를 marginalize함으로써 각 성분 귀속을 모르더라도 전체 분포를 학습할 수 있다.
Q15 Nested Cross-Validation(결합 교차검증)의 구조를 도식적으로 설명하고, inner loop와 outer loop의 각각의 역할과 사용하는 데이터 집합을 명확히 구분하라. 단순 CV와 Nested CV를 사용해야 하는 경우를 구분하라. 비교 +
풀이
Nested CV 구조:
전체 데이터 D = Training pool + Test set (Outer split, 한 번만 나눔)
Training pool → [Inner loop: K-fold CV] → 후보 모델 각각에 대해 CV 수행 → 가장 낮은 val error의 하이퍼파라미터 λ* 선택 Training pool 전체로 λ*로 재학습 Test set → 최종 generalization error 평가 (딱 한 번만 사용)
Inner loop 역할: 하이퍼파라미터 선택(모델 선택). 사용 데이터: Training pool의 K-fold.
Outer loop 역할: 최종 성능 평가(일반화 오차 추정). 사용 데이터: 별도 Test set.

단순 CV vs Nested CV 사용 구분:
• 단순 CV: 고정된 하이퍼파라미터로 generalization error만 추정할 때.
• Nested CV: 하이퍼파라미터 선택(모델 선택)과 최종 성능 평가를 동시에 수행할 때. Test set이 한 번만 쓰여야 공정한 평가 가능.
Q16 Bias-Variance Trade-off 관점에서 정규화 파라미터 λ가 모델에 미치는 영향을 설명하라. λ → 0 및 λ → ∞ 의 극한에서 각각 어떤 문제가 발생하는가? 최적 λ를 찾는 방법을 서술하라. 계산 +
풀이
Bias-Variance Trade-off with λ:
Expected Risk = Bias² + Variance + Noise
λ 증가 시:
• 파라미터 크기 ‖θ‖에 강한 페널티 → θ가 0 방향으로 수축(shrinkage)
• 모델 복잡도 감소 → Variance ↓ (다른 훈련 데이터에 덜 민감)
• 그러나 진짜 패턴도 제한 → Bias ↑

λ → 0 극한: 정규화 없음. 모델이 훈련 데이터 노이즈까지 학습 → Overfitting (High Variance, Low Bias).

λ → ∞ 극한: 페널티가 압도적 → θ* → 0 (모든 예측이 상수). 데이터를 무시 → Underfitting (Low Variance, High Bias).

최적 λ 탐색 방법:
① K-fold Cross-Validation: 여러 λ 후보에 대해 CV를 수행하고, validation error가 최소인 λ 선택.
② Grid Search 또는 Bayesian Optimization으로 λ 후보 탐색 범위 설정.
