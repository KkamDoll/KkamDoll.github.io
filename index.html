🔴 Stack — 07강

LIFO · 배열 기반

ArrayStack 구현

push · pop · top — 배열(리스트)로 스택 구현
쉬움

📐 동작 원리

  • 저장소: self.data = [] 하나로 모든 걸 처리. 리스트의 맨 끝 = 스택의 top.
  • push: append(item)으로 맨 뒤에 추가. O(1) 분할상환.
  • pop: 비어있으면 Exception. data.pop()으로 맨 뒤 제거 후 반환. O(1).
  • top: data[-1]로 제거 없이 맨 위만 확인. 파이썬의 음수 인덱스 활용.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
class ArrayStack:
    def __init__(self):
        self.data = []                  # 저장소: 빈 리스트

    def __len__(self): return len(self.data)
    def is_empty(self): return len(self.data) == 0

    def push(self, item):
        self.data.append(item)         # 맨 뒤 = top

    def pop(self):
        if self.is_empty(): raise Exception('Data is empty')
        return self.data.pop()          # 맨 뒤 제거 + 반환

    def top(self):
        return self.data[-1]            # 제거 없이 맨 위만

🧠 암기 포인트

  • push = append, pop = pop(), top = [-1] — 세 개 세트로 외우기
  • pop만 예외처리 있음. top은 예외처리 없이 그냥 [-1]

parentheses_matching

괄호가 올바르게 매칭됐는지 검사
쉬움

📐 동작 원리

  • 여는 괄호 (: 스택에 push. 나중에 닫힐 때 짝을 맞출 것임.
  • 닫는 괄호 ): 스택이 비어있으면 짝 없음 → 즉시 return False. 아니면 pop으로 여는 괄호 소비.
  • 최종 반환: return s.is_empty() — 스택이 비어있어야 모든 괄호가 짝 맞음. 남은 게 있으면 False.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def parantheses_matching(expr: str) -> bool:
    s = ArrayStack()
    for char in expr:
        if char == '(':
            s.push(char)
        elif char == ')':           # elif 주의! (if 아님)
            if s.is_empty(): return False
            s.pop()
    return s.is_empty()           # 스택이 비어야 True

🐛 Problems 파일 vs Answers 파일 비교

❌ problems 파일 (버그)
if char == '(': s.push(char)
if char == ')':     # if → elif 아님!
    ...
✅ answers 파일 (정답)
if char == '(': s.push(char)
elif char == ')':  # elif가 맞음
    ...
  • if로 쓰면 ')'도 무조건 체크되어 '('를 push한 직후 다시 pop할 수 있음
  • 반드시 elif로 써야 둘 중 하나만 실행됨

⚠️ 가장 많이 틀리는 부분

  • 마지막 return True ← 절대 안 됨! 반드시 return s.is_empty()
  • "(((" 처럼 열리기만 한 경우: 스택에 남아있어서 is_empty() = False → return False

🟠 ArrayQueue — 08강

원형 배열 · FIFO

핵심 구조 이해 — 원형(Circular) 배열

📐 왜 원형 배열인가?

  • 문제: 일반 배열에서 dequeue할 때마다 앞 원소를 지우면 나머지를 다 앞으로 당겨야 함 → O(n)
  • 해결: _front 포인터를 이동시키고 인덱스를 % len으로 순환시킴 → O(1)
  • 3개 변수: _data(배열), _front(시작 인덱스), _size(현재 원소 수)
  • 핵심 공식: 다음 빈 자리 = (front + size) % len. 다음 front = (front + 1) % len

first / dequeue

첫 원소 조회 / 제거하며 반환
중간

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def first(self):
    if self.is_empty(): raise Exception('Queue is empty')
    return self._data[self._front]       # _front 위치 그냥 반환

def dequeue(self):
    if self.is_empty(): raise Exception('Queue is empty')
    answer = self._data[self._front]
    self._data[self._front] = None         # 메모리 정리
    self._front = (self._front + 1) % len(self._data)  # 원형 이동
    self._size -= 1
    return answer

❌ 버그
return self._data[self.front]  # _ 빠짐
self.front = (self._front+1)%...  # _ 빠짐
✅ 정답
return self._data[self._front]
self._front = (self._front+1)%...

enqueue

원소 추가 — 빈 자리 계산이 핵심
중간

📐 동작 원리

  • 꽉 찼는지 확인: self._size == len(self._data)이면 resize(2배)
  • 빈 자리 계산: avail = (self._front + self._size) % len(self._data) — front에서 size만큼 앞으로
  • 삽입: self._data[avail] = e. size 증가.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def enqueue(self, e):
    if self._size == len(self._data):
        self._resize(2 * len(self._data))   # 꽉 차면 2배 확장
    avail = (self._front + self._size) % len(self._data)  # 다음 빈 자리
    self._data[avail] = e
    self._size += 1

🐛 Problems 파일 버그 — avail 계산 완전히 틀림

❌ 버그 3개!
if len(self._data) == len(self._size):  # _size는 int!
    self.resize(...)                       # _resize 아님
avail = (self._data + 1) % len(self.data) # 완전히 틀림
✅ 정답
if self._size == len(self._data):
    self._resize(2 * len(self._data))
avail = (self._front + self._size) % len(self._data)

⚠️ 핵심 함정

  • avail = (front + 1) ← 틀림. 반드시 (front + size). size=0이면 front 위치가 빈 자리임!

_resize

배열 크기 변경 — 원형 순서를 유지하며 복사
중간

📐 동작 원리

  • old 저장: 기존 배열을 old에 백업. 새 배열 [None]*cap 생성.
  • walk = _front: 현재 큐의 시작 위치부터 순서대로 복사 시작.
  • 순환 복사: walk = (1 + walk) % len(old)로 old 배열을 원형으로 순회하며 새 배열 인덱스 0부터 순서대로 복사.
  • _front = 0: 새 배열에서는 인덱스 0부터 시작하므로 front 초기화.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def _resize(self, cap):
    old = self._data
    self._data = [None] * cap
    walk = self._front                  # 큐의 시작부터
    for k in range(self._size):
        self._data[k] = old[walk]
        walk = (1 + walk) % len(old)  # old 배열 원형 순회
    self._front = 0                    # 새 배열: front는 항상 0

🧠 암기 포인트

  • 복사할 때 len(old)로 나눔 (새 배열 아님!). 새 배열은 0부터 순서대로 채움.
  • 마지막에 self._front = 0 초기화 필수!

def _add_root(self, e):
    if self._root is not None:
        raise ValueError('Root exists')   # 루트가 이미 있으면 에러
    self._size = 1
    self._root = self._Node(e)
    return self._make_position(self._root)

def _add_left(self, p, e):
    node = self._validate(p)
    if node._left is not None:
        raise ValueError('Left child exists')
    self._size += 1
    node._left = self._Node(e, node)    # 부모 참조 설정 필수!
    return self._make_position(node._left)

🧠 암기 포인트

  • 두 함수 모두 존재 여부 먼저 확인 → ValueError → size 증가 → 노드 생성 → position 반환

_delete

자식이 0~1개인 노드 삭제
어려움

📐 동작 원리

  • 제약 확인: 자식이 2개면 ValueError. 이 함수는 자식 0~1개만 처리.
  • 자식 찾기: child = node._left if node._left else node._right. 둘 다 없으면 child = None.
  • 자식의 부모 갱신: child가 있으면 child._parent = node._parent (조부모로 연결).
  • 부모 갱신: 삭제 노드가 루트면 루트 교체. 아니면 부모의 left/right 포인터를 child로 교체.
  • 마무리: size--, node._parent = node(쓰레기 표시), element 반환.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def _delete(self, p):
    node = self._validate(p)
    if self.num_children(p) == 2:
        raise ValueError('Position has two children')

    child = node._left if node._left else node._right  # 자식 (없으면 None)
    if child is not None:
        child._parent = node._parent              # 자식의 부모 = 조부모
    if node is self._root:
        self._root = child
    else:
        parent = node._parent
        if node is parent._left:
            parent._left = child
        else:
            parent._right = child
    self._size -= 1
    node._parent = node                           # deprecated 표시
    return node._element

calculate_node_depth

재귀로 depth 파라미터를 내려보내며 탐색
중간

📐 동작 원리 — depth를 파라미터로 전달하는 재귀

  • Base case ①: node가 None이면 return -1 (못 찾음)
  • Base case ②: node.key == key이면 현재 depth 반환
  • 왼쪽 재귀: depth+1로 호출. 결과가 -1이 아니면 찾은 것 → 바로 반환
  • 오른쪽 재귀: 결과 그대로 반환

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def calculate_node_depth(tree, key):
    def _calc(node, key, depth):
        if node is None: return -1          # 못 찾음
        if node.key == key: return depth    # 찾음: 현재 깊이 반환

        left = _calc(node.left, key, depth + 1)
        if left != -1: return left           # 왼쪽에서 찾았으면 즉시 반환
        return _calc(node.right, key, depth + 1)

    return _calc(tree.root, key, 0)

🧠 암기 포인트

  • b3의 depth 함수(parent 타고 올라가기)와 다른 접근법: 여기선 재귀 내려갈 때 depth를 함께 넘김
  • 못 찾으면 -1, 찾으면 depth — != -1 체크로 조기 종료

sum_subtree

특정 키의 서브트리 value 합산 — 2단계 재귀
중간

📐 동작 원리 — 2단계

  • [1단계] _find_node: key로 노드 탐색. 없으면 return -1.
  • [2단계] _sum_subtree: 찾은 노드부터 서브트리의 value 합산 (key가 아닌 value!).
  • 공식: node.value + _sum(왼) + _sum(오)

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def sum_subtree(tree, key):
    def _find_node(node, key):            # 1단계: 노드 찾기
        if node is None: return None
        if node.key == key: return node
        left = _find_node(node.left, key)
        if left is not None: return left
        return _find_node(node.right, key)

    def _sum_subtree(node):               # 2단계: 서브트리 합산
        if node is None: return 0
        return node.value + _sum_subtree(node.left) + _sum_subtree(node.right)

    node = _find_node(tree.root, key)
    if node is None: return -1
    return _sum_subtree(node)

⚠️ 함정

  • 탐색은 key로, 합산은 value로. 둘을 혼동하면 오답!

EulerTour._tour

전위+후위를 결합한 트리 순회 템플릿
어려움

📐 동작 원리

  • 전위 hook: _hook_previsit(p, d, path) — 자식 방문 전에 실행
  • 자식 순회: for c in children(p)로 각 자식에 재귀. d+1로 깊이 증가.
  • path 관리: 자식 방문 시 path.append(0), 다음 자식마다 path[-1] += 1, 끝나면 path.pop()
  • 후위 hook: _hook_postvisit(p, d, path, results) — 자식 방문 후 실행. 반환값이 최종 결과.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def _tour(self, p, d, path):
    self._hook_previsit(p, d, path)     # 전위: 자식 전에 실행
    results = []
    path.append(0)

    for c in self._tree.children(p):
        results.append(self._tour(c, d+1, path))
        path[-1] += 1                   # 다음 자식 인덱스로
    path.pop()

    answer = self._hook_postvisit(p, d, path, results)  # 후위: 자식 후에 실행
    return answer

🧠 암기 포인트

  • previsit(전위) → 자식 순회 → postvisit(후위) 순서
  • path[-1] += 1은 for 루프 안에서 자식 방문 후에 실행
  • hook 메서드들은 기본적으로 pass. 서브클래스에서 override해서 활용

Unsorted vs Sorted 비교

📐 핵심 트레이드오프

UnsortedPriorityQueue

  • add: O(1) — 그냥 맨 뒤에 추가

UnsortedPriorityQueue._find_min

전체 탐색으로 최솟값 위치(Position) 찾기
중간

📐 동작 원리

  • 초기화: small = self._data.first()로 첫 번째를 최솟값 후보로.
  • 순회: walk = after(small)에서 시작해 끝까지 탐색.
  • 비교: walk.element() < small.element()이면 small 업데이트.
  • 반환: 최솟값의 Position 반환 (값이 아닌 위치!). min()과 remove_min()에서 사용.

def _find_min(self):
    if self.is_empty(): raise Exception('Priority queue is empty')
    small = self._data.first()
    walk = self._data.after(small)
    while walk is not None:
        if walk.element() < small.element():  # walk가 더 작으면
            small = walk
        walk = self._data.after(walk)
    return small                            # Position 반환

SortedPriorityQueue.add

정렬 순서를 유지하며 올바른 위치에 삽입
중간

📐 동작 원리

  • new item 생성: newest = self._Item(key, value)
  • 뒤에서부터 탐색: walk = self._data.last()에서 시작해 before()로 앞으로 이동.
  • 이동 조건: newest < walk.element()이면 계속 앞으로. (새 아이템이 현재보다 작으면 더 앞에 있어야 함)
  • 삽입: walk가 None이면 맨 앞에 add_first, 아니면 walk 뒤에 add_after.

💻 코드

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def add(self, key, value):
    newest = self._Item(key, value)
    walk = self._data.last()              # 뒤에서 시작
    while walk is not None and newest < walk.element():
        walk = self._data.before(walk)   # 앞으로 이동
    if walk is None:
        self._data.add_first(newest)       # 모두보다 작음 → 맨 앞
    else:
        self._data.add_after(walk, newest)

❌ 버그

# before(walk) 가 맞음
✅ 정답

def heap_sort(arr):

def preorder(tree):
    result = []
    def _recur(node):
        if node is not None:
            result.append(node.key)   # ① 나 먼저 (Pre = 먼저)
            _recur(node.left)
            _recur(node.right)
    _recur(tree.root())
    return result

def postorder(tree):
    result = []
    def _recur(node):
        if node is not None:
            _recur(node.left)
            _recur(node.right)
            result.append(node.key)   # ③ 나 나중에 (Post = 나중)
    _recur(tree.root())
    return result

🧠 preorder vs postorder

b3 · depth(tree, key) → int

find_node 후 parent 따라 올라가며 카운트
중간 · 8점
🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def depth(tree, key):
    def find_node(node, key):
        if node is None: return None
        if node.key == key: return node
        left = find_node(node.left, key)
        if left is not None: return left
        return find_node(node.right, key)

    target = find_node(tree.root(), key)
    if target is None: return -1

    d, curr = 0, target
    while curr.parent is not None:
        curr = curr.parent
        d += 1
    return d

b4 · uncle(tree, key) → int

삼촌 = 조부모의 반대쪽 자식. 없으면 -1
중간 · 8점
🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def uncle(tree, key):
    def find_node(node, key):
        if node is None: return None
        if node.key == key: return node
        left = find_node(node.left, key)
        if left is not None: return left
        return find_node(node.right, key)

    target = find_node(tree.root(), key)
    p = target.parent
    if p is None: return -1            # target이 루트
    g = p.parent
    if g is None: return -1            # p가 루트

    if g.left == p:
        return g.right.key if g.right is not None else -1
    else:
        return g.left.key if g.left is not None else -1

🧠 -1 반환 케이스 3가지

  • ① p is None (target=루트) ② g is None (p=루트) ③ 반대쪽 자식이 None

b5 · max_path_sum(tree) → int

루트→리프 경로 중 key 합 최대값

🙈 코드를 가렸습니다버튼을 눌러 다시 보세요
def max_path_sum(tree):
    def _recur(node):
        if node is None: return 0        # 반드시 0! (-1 아님)
        left  = _recur(node.left)
        right = _recur(node.right)
        return node.key + max(left, right)

    if tree.root is None: return 0
    return _recur(tree.root())

⚠️ base case는 반드시 return 0

  • return -1로 쓰면 리프의 max(-1,-1)=-1이 더해져서 오답!

  • 괄호매칭: 열면 push, 닫으면 pop. 끝에 is_empty() 반환
  • elif 주의: if가 아닌 elif

  • enqueue avail = (front+size)%len (front+1 아님!)
  • resize: len(old)로 나눔. 끝에 _front=0

🔵 Heap / 시험 A

  • upheap: j>0 AND 나<부모 → swap → upheap(parent)
  • downheap: 왼가정→오비교→swap→downheap(small_child)
  • add: append → upheap(len-1)
  • remove_min: swap(0,끝)→pop→is_empty체크→downheap(0)
  • heap_sort: 전부 add → 전부 remove_min

🟢 Tree / 시험 B

  • preorder: append 앞 (나→왼→오)
  • postorder: append 뒤 (왼→오→나)
  • depth: find_node → parent 타고 올라가며 카운트
  • uncle: p=parent, g=g.parent, 반대쪽 자식. -1 케이스 3가지
  • max_path_sum: 나+max(왼,오). base case = 0

🟣 Priority Queue

  • Unsorted: add=O(1), find_min=O(n) 전탐색
  • Sorted: add=O(n) 뒤→앞 탐색, min=O(1)
  • find_min: first()에서 시작, after()로 순회, return small 필수
  • sorted.add: last()→before(walk). before(walk), 인자 필수
