01
데이터 벡터화
Data Vectorization / Feature Engineering
MML §8.1

현실의 데이터는 그 자체로는 기계학습 알고리즘에 바로 쓸 수 없다. 반드시 실수 벡터로 가공해야 한다. 이 과정에서 얻어지는 각 차원의 값을 feature, attribute, covariate 등으로 부른다.

핵심 원칙
원시 데이터 → 전처리 → 실수 벡터 \(\mathbf{x}_i \in \mathbb{R}^D\). 이후 기계학습 알고리즘에 투입.
  • 이름(Name)의미 없음 → 삭제
  • 성별(Gender)범주형 → 수치 레이블 (예: M=−1, F=+1)
  • 학위(Degree)순서형 → Leveling (BSc=1, MSc=2, PhD=3 등)
  • 우편번호(Postcode)지역 코드 → 위도/경도로 변환
  • 나이, 연봉이미 실수 → 그대로 사용 (필요시 정규화)

Function Model vs. Probabilistic Model
MML §8.1

같은 입력 데이터에 대해 출력을 어떻게 정의하느냐에 따라 두 종류의 모델이 존재한다.

"소득 수준은 연령과 관계 있다" → \(x\): 연령, \(y\): 소득, 모델: \(y=f(x)\)
같은 \(x=60\)세에 대해 함수 모델은 단일 값, 확률 모델은 분포(예: Gaussian)를 반환한다.
03
학습의 구성
ML Pipeline: Learning = Parameter Finding
MML §8.2

기계학습 알고리즘은 일반적으로 세 단계로 구성된다.

  • ① 예측/추론학습된 모델에 새 \(\mathbf{x}\)를 넣어 \(\hat{y}\) 또는 \(p(y\mid\mathbf{x})\)를 계산한다.
  • ② 파라미터 추정주어진 데이터로 모델 파라미터 \(\boldsymbol{\theta}\)를 최적화한다. (Training)
  • ③ 모델 선택어떤 형태의 함수·모델이 적합한지 결정한다. (Hyperparameter Tuning)
파라미터 vs. 하이퍼파라미터
파라미터 \(\boldsymbol{\theta}\): 데이터로부터 학습되는 값. 예: 선형회귀의 기울기 \(a\), 절편 \(b\).
하이퍼파라미터: 사람이 미리 정하는 값. 예: 정규화 강도 \(\lambda\), 다항식 차수 \(d\). 교차검증으로 선택.
04
경험적 위험 최소화
Empirical Risk Minimization (ERM)
MML §8.2

예측인자(predictor)가 함수형일 때, 데이터셋 전체의 평균 오차를 경험적 위험(Empirical Risk)으로 정의하고 이를 최소화하는 파라미터를 찾는다.

Loss Function
\[\ell(y_n,\,\hat{y}_n)\]
실제값과 예측값의 차이를 non-negative 값으로 변환한 함수
Empirical Risk — MML Eq. 8.6
\[\mathbf{R}_{\text{emp}}(f,\,\mathbf{X},\,\mathbf{y}) \;=\; \frac{1}{N}\sum_{n=1}^{N} \ell\!\left(y_n,\;\hat{y}_n\right)\]
Least-Squares Loss (예제 8.2) — MML Eq. 8.7–8.9
\[\ell(y_n,\hat{y}_n) = (y_n - \hat{y}_n)^2\]
squared loss
\[\min_{\boldsymbol{\theta}\in\mathbb{R}^D}\; \frac{1}{N}\sum_{n=1}^{N}\!\left(y_n - \boldsymbol{\theta}^\top\mathbf{x}_n\right)^{\!2}\]
선형 예측자 \(\hat{y}_n=\boldsymbol{\theta}^\top\mathbf{x}_n\) 사용
\[= \;\min_{\boldsymbol{\theta}}\; \frac{1}{N}\left\|\mathbf{y} - \mathbf{X}\boldsymbol{\theta}\right\|^2\]
행렬 형태 (Eq. 8.9)
Expected (True) Risk — 학습의 진짜 목표
\[\mathbf{R}_{\text{true}}(f) \;=\; \mathbb{E}_{\mathbf{x},\,y}\!\left[\ell\!\left(y,\, f(\mathbf{x})\right)\right]\]
미래 데이터에도 잘 동작하는 함수를 원하므로, 실제 분포 전체에 대한 기댓값을 최소화하는 것이 목표

학습 데이터의 i.i.d. 가정: 훈련 데이터 \((\mathbf{x}_1,y_1),\ldots,(\mathbf{x}_N,y_N)\)는 독립이고 동일한 분포(i.i.d.)에서 추출되었다고 가정한다.

05
오버피팅 & 정규화
Overfitting & Regularization
MML §8.2.3

오버피팅(Overfitting): 훈련 데이터에는 매우 낮은 loss를 보이지만, 새로운 데이터에서 expected risk가 크게 증가하는 현상.

세 가지 피팅 상태
Underfitting: 모델이 너무 단순 → 훈련/테스트 모두 error 높음.
Good Fitting: 적절한 복잡도 → 일반화 잘 됨.
Overfitting: 모델이 너무 복잡 → 훈련 error↓, 테스트 error↑.

해결책: 정규화(Regularization) — 손실 함수에 페널티 항을 추가하여 파라미터가 너무 커지는 것을 억제한다.

Regularized Least-Squares (예제 8.3) — MML Eq. 8.11–8.12
\[\text{기본:}\quad \min_{\boldsymbol{\theta}}\; \frac{1}{N}\left\|\mathbf{y}-\mathbf{X}\boldsymbol{\theta}\right\|^2\]
\[\text{정규화:}\quad \min_{\boldsymbol{\theta}}\; \frac{1}{N}\left\|\mathbf{y}-\mathbf{X}\boldsymbol{\theta}\right\|^2 + \lambda\left\|\boldsymbol{\theta}\right\|^2\]
\(\lambda\): 정규화 파라미터 (하이퍼파라미터)
\(\lambda\)가 클수록 파라미터 크기에 더 큰 페널티

추가 항 \(\|\boldsymbol{\theta}\|^2\)를 regularizer라 한다. 손실 최소화와 파라미터 크기 최소화 간의 trade-off가 \(\lambda\)로 조절된다.

06
교차검증
K-fold Cross-Validation
MML §8.2.4

미래 데이터의 성능을 추정하기 위해, 보유한 데이터를 K개의 chunk로 나눠 K−1개를 훈련에, 1개를 검증에 사용하는 방법. 이를 K번 반복하여 평균 위험을 계산한다.

K-fold Generalization Error 추정
\[\mathbb{E}_{\mathcal{V}}\!\left[R(f,\mathcal{V})\right] \;\approx\; \frac{1}{K}\sum_{k=1}^{K} R\!\left(f^{(k)},\,\mathcal{V}^{(k)}\right)\]
  • \(\mathcal{R}^{(k)}\) (Training)\(k\)번째 fold를 제외한 나머지로 모델 \(f^{(k)}\) 학습
  • \(\mathcal{V}^{(k)}\) (Validation)\(k\)번째 fold → 실제적 위험 \(R(f^{(k)}, \mathcal{V}^{(k)})\) 계산
  • \(\mathcal{R}\cup\mathcal{V}=\mathcal{D},\;\mathcal{R}\cap\mathcal{V}=\emptyset\)모든 데이터를 다 활용, 겹치지 않음
결합 교차검증 (Nested CV) — 모델 선택용
Inner loop: training + validation으로 각 후보 모델의 empirical risk 기댓값 추정 → 최적 모델 선택.
Outer loop: 별도 test data로 최종 모델의 generalization error 평가.

\(\mathbb{E}_{\mathcal{V}}\!\left[\mathbf{R}(\mathcal{V}\mid M)\right] \approx \dfrac{1}{K}\displaystyle\sum_{k=1}^{K} \mathbf{R}\!\left(\mathcal{V}^{(k)}\mid M\right)\)

단점: K번 반복 학습이 필요하므로 계산 비용이 매우 크다. 최근에는 병렬 컴퓨팅으로 해결.

07
최대 우도 추정
Maximum Likelihood Estimation (MLE)
MML §8.3

관측된 데이터를 가장 잘 설명하는 파라미터 \(\boldsymbol{\theta}\)를 찾는 방법. i.i.d. 가정 하에 결합 우도를 곱 형태로 분해한 뒤, 음의 로그 우도(negative log-likelihood)를 최소화한다.

Likelihood (i.i.d. 가정)
\[p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta}) \;=\; \prod_{n=1}^{N} p\!\left(y_n\mid\mathbf{x}_n,\boldsymbol{\theta}\right)\]
Negative Log-Likelihood (NLL)
\[\mathcal{L}(\boldsymbol{\theta}) = -\log p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta}) = -\sum_{n=1}^{N}\log p\!\left(y_n\mid\mathbf{x}_n,\boldsymbol{\theta}\right)\]
최대화 → 음수 취해 최소화 문제로 변환 · 곱 → 합 (수치 안정성)
Gaussian Likelihood → Least Squares (예제 8.5) — MML Eq. 8.15, 8.18
\[p(y_n\mid\mathbf{x}_n,\boldsymbol{\theta}) = \mathcal{N}\!\left(y_n\;\middle|\;\mathbf{x}_n^\top\boldsymbol{\theta},\;\sigma^2\right)\]
\[\mathcal{L}(\boldsymbol{\theta}) = \frac{1}{2\sigma^2}\sum_{n=1}^{N}\!\left(y_n - \mathbf{x}_n^\top\boldsymbol{\theta}\right)^{\!2} - N\log\frac{1}{\sqrt{2\pi\sigma^2}}\]
\(\sigma\) 상수이면
두 번째 항은 \(\boldsymbol{\theta}\)와 무관
→ NLL 최소화 = 최소자승
MLE의 특성과 한계
Asymptotic Consistency: \(N\to\infty\)일 때 오차 분산이 0에 수렴.
Small data overfitting: 데이터가 적으면 overfitting 발생.
사전 지식 반영 불가: \(\boldsymbol{\theta}\)에 대한 사전 정보를 활용하지 않는다.
08
최대 사후 추정
Maximum A Posteriori (MAP)
MML §8.3

MLE에 사전 분포(prior) \(p(\boldsymbol{\theta})\)를 추가하여 \(\boldsymbol{\theta}\)에 대한 사전 지식을 반영하는 방법. 베이즈 정리를 통해 후위 확률(posterior)을 최대화한다.

Bayes' Theorem
\[p(\boldsymbol{\theta}\mid\mathbf{x}) = \frac{p(\mathbf{x}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})}{p(\mathbf{x})}\]
\[p(\boldsymbol{\theta}\mid\mathbf{x}) \;\propto\; p(\mathbf{x}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})\]
\(p(\mathbf{x})\)는 \(\boldsymbol{\theta}\)와 무관 → 비례 관계로 단순화
MAP Estimate
\[\boldsymbol{\theta}_{\text{MAP}} = \operatorname*{arg\,max}_{\boldsymbol{\theta}}\;\Bigl[\log p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta}) + \log p(\boldsymbol{\theta})\Bigr]\]
Gaussian Prior → L2 Regularization (예제 8.6) — MML §8.3.2
\[p(\boldsymbol{\theta}) = \mathcal{N}(\mathbf{0},\,\boldsymbol{\Sigma})\]
등방성: \(\boldsymbol{\Sigma}=\frac{1}{\lambda}\mathbf{I}\)
\[\Rightarrow\quad \boldsymbol{\theta}_{\text{MAP}} = \operatorname*{arg\,min}_{\boldsymbol{\theta}}\;\frac{1}{N}\left\|\mathbf{y}-\mathbf{X}\boldsymbol{\theta}\right\|^2 + \lambda\left\|\boldsymbol{\theta}\right\|^2\]
Gaussian prior MAP
≡ L2 정규화(Ridge)
MLE vs. MAP
MLE: \(p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta})\) 최대화. 사전 지식 없음. 한 점 추정.
MAP: \(p(\boldsymbol{\theta}\mid\mathcal{X},\mathcal{Y})\) 최대화. 사전 분포 \(p(\boldsymbol{\theta})\) 활용. 한 점 추정.
두 방법 모두 point estimate이다.
09
베이지안 추론
Bayesian Inference
MML §8.4

MLE/MAP가 \(\boldsymbol{\theta}\)의 한 점을 추정하는 것과 달리, 베이지안 추론은 데이터 전체에 대해 파라미터 분포 \(p(\boldsymbol{\theta}\mid\mathcal{X})\) 전체를 추정한다.

Posterior Distribution
\[p(\boldsymbol{\theta}\mid\mathcal{X}) = \frac{p(\mathcal{X}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})}{p(\mathcal{X})}\]
\[p(\mathcal{X}) = \int p(\mathcal{X}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})\;d\boldsymbol{\theta}\]
marginal likelihood (evidence)
대부분 closed-form 불가
Predictive Distribution
\[p(\mathbf{x}) = \int p(\mathbf{x}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})\;d\boldsymbol{\theta} = \mathbb{E}_{\boldsymbol{\theta}}\!\left[p(\mathbf{x}\mid\boldsymbol{\theta})\right]\]
가능한 모든 \(\boldsymbol{\theta}\)에 대한 평균 → 파라미터에 종속되지 않음
방법
계산
결과
MLE
최적화 (미분=0)
한 점 \(\boldsymbol{\theta}^*\)
MAP
최적화 (prior 포함)
한 점 \(\boldsymbol{\theta}_{\text{MAP}}\)
Bayesian
적분 (posterior 전체)
분포 \(p(\boldsymbol{\theta}\mid\mathcal{X})\)
베이지안 추론의 특징
장점: 파라미터 불확실성(uncertainty) 정량화 가능, 사전 지식 자연스럽게 반영.
단점: \(p(\mathcal{X})\) 계산을 위한 적분이 대부분 closed-form으로 불가 → 켤레 분포(conjugate prior) 아니면 수치 근사(Monte Carlo 등) 필요.
10
잠재 변인 모델
Latent Variable Model
MML §8.4

관측 데이터 \(\mathbf{x}\) 외에 숨겨진 변인(latent variable) \(\mathbf{z}\)가 존재한다고 가정하여, 모델의 표현력을 높이면서도 파라미터 추정을 가능하게 하는 방법.

z를 주변화(marginalize)하여 우도 계산
\[p(\mathbf{x}\mid\boldsymbol{\theta}) = \int p(\mathbf{x}\mid\mathbf{z},\boldsymbol{\theta})\;p(\mathbf{z})\;d\mathbf{z}\]
\(\mathbf{z}\)의 prior \(p(\mathbf{z})\)가 주어지면 적분으로 \(\mathbf{z}\) 제거 → 우도는 \(\boldsymbol{\theta}\)만의 함수
z에 대한 Posterior
\[p(\mathbf{z}\mid\mathbf{x},\boldsymbol{\theta}) = \frac{p(\mathbf{x}\mid\mathbf{z},\boldsymbol{\theta})\;p(\mathbf{z})}{p(\mathbf{x}\mid\boldsymbol{\theta})}\]
θ에 대한 Posterior (Bayesian)
\[p(\boldsymbol{\theta}\mid\mathcal{X}) = \frac{p(\mathcal{X}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta})}{p(\mathcal{X})}\]
\[p(\mathcal{X}\mid\boldsymbol{\theta}) = \int p(\mathcal{X}\mid\mathbf{z},\boldsymbol{\theta})\;p(\mathbf{z})\;d\mathbf{z}\]
활용 및 계산 조건
10장(PCA), 11장(Gaussian Mixture Model), Hidden Markov Model 등. \(\mathbf{z}\)가 켤레 prior(conjugate prior)여야 closed-form 계산 가능. 아닌 경우 MCMC, Variational Inference 등 수치해석 근사 필요.
11
모델 선택 & 베이즈 요인
Model Selection & Bayes Factor
MML §8.6

오캄의 면도날(Occam's Razor): 데이터를 가장 간단하게 설명하는 모델이 overfitting 위험이 낮다. 모델 선택은 "획득한 데이터를 가장 쉽게 설명하는 가설을 찾는 과정"이다.

베이지안 모델 선택 — MML §8.6
\[p(\mathcal{M}_k\mid\mathcal{D}) \;\propto\; p(\mathcal{M}_k)\;p(\mathcal{D}\mid\mathcal{M}_k)\]
\[p(\mathcal{D}\mid\mathcal{M}_k) = \int p(\mathcal{D}\mid\boldsymbol{\theta}_k)\;p(\boldsymbol{\theta}_k\mid\mathcal{M}_k)\;d\boldsymbol{\theta}_k\]
주변 우도 (Marginal Likelihood)
\[M^* = \operatorname*{arg\,max}_{\mathcal{M}_k}\; p(\mathcal{M}_k\mid\mathcal{D})\]
Bayes Factor (모델 비교)
\[\underbrace{\dfrac{p(\mathcal{M}_1\mid\mathcal{D})}{p(\mathcal{M}_2\mid\mathcal{D})}}_{\text{posterior odds}} \;=\; \underbrace{\dfrac{p(\mathcal{M}_1)}{p(\mathcal{M}_2)}}_{\text{prior odds}} \;\times\; \underbrace{\dfrac{p(\mathcal{D}\mid\mathcal{M}_1)}{p(\mathcal{D}\mid\mathcal{M}_2)}}_{\text{Bayes factor}}\]
  • Prior Odds사전 확률 비 — \(\mathcal{M}_1\)이 \(\mathcal{M}_2\)보다 사전에 얼마나 더 개연성 있는지
  • Bayes Factor우도 비 — 데이터 \(\mathcal{D}\)가 어느 모델에서 더 잘 설명되는지
  • 결론BF > 1이면 \(\mathcal{M}_1\) 선택. uniform prior이면 BF = 주변 우도 비
주변 우도의 역할 (오캄의 면도날)
주변 우도는 Model complexity와 Data fit 사이의 trade-off를 자동으로 반영한다. 복잡한 모델은 prior가 넓게 퍼져 희석되어 marginal likelihood가 작아진다 (Jeffreys-Lindley 역설).

Q1현실의 데이터를 기계학습에 적용하기 위해 필요한 전처리 과정을 설명하고, 범주형 변수(categorical variable)를 수치화하는 방법 두 가지를 제시하라.개념+
풀이
현실 데이터는 문자, 범주 등 다양한 형태이므로, 기계학습 알고리즘 적용 전 반드시 실수 벡터로 변환해야 한다. 이를 feature engineering이라 한다.

① Label Encoding: 각 범주에 정수를 부여 (예: M=−1, F=+1). 순서가 없는 명목형에는 잘못된 순서 관계를 부여할 수 있어 주의 필요.
② One-hot Encoding: K개 범주를 K차원 이진 벡터로 표현. 순서 없는 범주형에 적합하지만 차원이 늘어남.
Q2함수 모델과 확률 분포 모델의 차이를 설명하고, 각각이 어떤 학습 방법론에 대응하는지 서술하라.비교+
풀이
함수 모델 \(f:\mathbb{R}^D\to\mathbb{R}\): 입력 \(\mathbf{x}\)에 대해 하나의 결정론적 값 \(\hat{y}\)을 반환. 불확실성 표현 불가. → ERM으로 파라미터 추정.

확률 분포 모델: 같은 입력에 대해 \(p(y\mid\mathbf{x},\boldsymbol{\theta})\)라는 확률 분포를 반환. 예측의 불확실성을 정량화할 수 있음. → MLE / MAP / Bayesian Inference로 파라미터 추정.
Q3기계학습 알고리즘의 세 단계(예측/추론, 파라미터 추정, 모델 선택)를 각각 설명하고, 파라미터와 하이퍼파라미터의 차이를 예와 함께 서술하라.개념+
풀이
① 예측/추론: 학습된 모델 \(f(\mathbf{x},\boldsymbol{\theta}^*)\)에 새 \(\mathbf{x}\)를 넣어 \(\hat{y}\)을 얻는 단계.
② 파라미터 추정(Training): 주어진 훈련 데이터로 손실 함수를 최소화하는 \(\boldsymbol{\theta}^*\)를 찾는 단계.
③ 모델 선택: 함수의 형태와 하이퍼파라미터를 결정하는 단계. Nested Cross-Validation 사용.

파라미터: 데이터로부터 학습. 예: 선형회귀 \(y=ax+b\)에서 \(a,b\).
하이퍼파라미터: 사람이 사전에 설정. 예: 정규화 강도 \(\lambda\), 다항식 차수 \(d\).
Q4N=3개의 데이터 포인트 (1,2), (2,4), (3,5)에 대해 선형 예측자 \(\hat{y}=\theta x\)를 사용할 때, 경험적 위험(least-squares loss)을 \(\theta\)의 함수로 표현하고 최적 \(\theta^*\)를 구하라.계산+
풀이
\[R_{\text{emp}}(\theta) = \frac{1}{3}\left[(2-\theta)^2 + (4-2\theta)^2 + (5-3\theta)^2\right] = \frac{1}{3}\left[14\theta^2 - 50\theta + 45\right]\]
최소화 조건 \(\dfrac{dR}{d\theta}=0\):
\[\frac{1}{3}(28\theta-50)=0 \quad\Rightarrow\quad \theta^* = \frac{50}{28} = \frac{25}{14} \approx 1.786\]
Q5경험적 위험(empirical risk)과 예측 위험(expected risk)의 차이를 설명하고, 오버피팅이 발생하면 두 값이 어떤 관계를 갖는지 서술하라.개념+
풀이
경험적 위험: 보유한 훈련 데이터에 대한 평균 손실.
\[\mathbf{R}_{\text{emp}}(f,\mathbf{X},\mathbf{y}) = \frac{1}{N}\sum_{n=1}^N \ell(y_n,f(\mathbf{x}_n))\]
예측 위험: 실제 데이터 분포 전체에 대한 기대 손실.
\[\mathbf{R}_{\text{true}}(f) = \mathbb{E}_{\mathbf{x},y}\!\left[\ell(y,f(\mathbf{x}))\right]\]
오버피팅 시:
\[\mathbf{R}_{\text{emp}}(f,\mathbf{X}_{\text{train}},\mathbf{y}_{\text{train}}) \;\ll\; \mathbf{R}_{\text{true}}(f)\]
즉, 훈련 데이터에서의 성능이 실제 성능을 크게 과소추정(underestimates)하게 된다.
Q6최소자승 문제 \(\min_\theta \frac{1}{N}\|\mathbf{y}-\mathbf{X}\boldsymbol\theta\|^2\)에 L2 정규화 \(\lambda\|\boldsymbol\theta\|^2\)를 추가했을 때, 최적해 \(\boldsymbol\theta^*\)를 closed-form으로 구하라.계산+
풀이
목적함수: \(J(\boldsymbol{\theta}) = \frac{1}{N}\|\mathbf{y}-\mathbf{X}\boldsymbol{\theta}\|^2 + \lambda\|\boldsymbol{\theta}\|^2\). 미분하여 0으로 놓기:
\[\nabla_{\boldsymbol{\theta}} J = \frac{2}{N}\mathbf{X}^\top(\mathbf{X}\boldsymbol{\theta}-\mathbf{y}) + 2\lambda\boldsymbol{\theta} = \mathbf{0}\]
\[\boxed{\boldsymbol{\theta}^* = \left(\mathbf{X}^\top\mathbf{X} + N\lambda\mathbf{I}\right)^{-1}\mathbf{X}^\top\mathbf{y}}\]
\(\lambda>0\)이면 \(\mathbf{X}^\top\mathbf{X}\)가 singular여도 역행렬 존재 → 수치 안정성 확보.
Q7K-fold 교차검증의 절차를 설명하고, 단순 train/test split보다 유리한 이유를 서술하라. K=5일 때 학습 횟수는?개념+
풀이
절차: ① 데이터를 K개 fold로 균등 분할. ② \(k=1,\ldots,K\)에 대해 \(k\)번째 fold를 validation set, 나머지를 training set으로 학습·평가. ③ K번 반복 후 평균을 generalization error 추정치로 사용.
\[\mathbb{E}_{\mathcal{V}}\!\left[R(f,\mathcal{V})\right] \approx \frac{1}{K}\sum_{k=1}^{K} R\!\left(f^{(k)},\mathcal{V}^{(k)}\right)\]
장점: 모든 데이터가 한 번씩 validation에 쓰이므로 추정 분산이 낮음.
K=5일 때: 학습 5회, 검증 5회.
Q8MLE에서 i.i.d. 가정 하에 결합 우도를 negative log-likelihood로 변환하는 과정을 유도하고, 이 변환이 수치적으로 편리한 이유를 설명하라.유도+
풀이
i.i.d. 가정 → 결합 우도:
\[p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta}) = \prod_{n=1}^{N} p(y_n\mid\mathbf{x}_n,\boldsymbol{\theta})\]
log 취하기 (단조증가 → argmax 동일):
\[\log p(\mathcal{Y}\mid\mathcal{X},\boldsymbol{\theta}) = \sum_{n=1}^{N}\log p(y_n\mid\mathbf{x}_n,\boldsymbol{\theta})\]
최대화 → 음수 취해 최소화:
\[\mathcal{L}(\boldsymbol{\theta}) = -\sum_{n=1}^{N}\log p(y_n\mid\mathbf{x}_n,\boldsymbol{\theta})\]
수치적 이유: 확률값 \(p\in(0,1]\)을 N번 곱하면 float underflow(수치 0) 발생 위험. log를 취하면 곱 → 합으로 변환되어 안정적 계산 가능.
Q9Gaussian 우도 \(p(y_n\mid\mathbf{x}_n,\boldsymbol\theta)=\mathcal{N}(y_n\mid\mathbf{x}_n^\top\boldsymbol\theta,\,\sigma^2)\)에서 MLE를 수행하면 최소자승 문제와 동일함을 유도하라. (MML 예제 8.5)유도+
풀이
Gaussian 분포 대입:
\[\mathcal{L}(\boldsymbol{\theta}) = \frac{1}{2\sigma^2}\sum_{n=1}^{N}(y_n-\mathbf{x}_n^\top\boldsymbol{\theta})^2 \;+\; N\log\sqrt{2\pi\sigma^2}\]
\(\sigma\)가 상수이면 두 번째 항은 \(\boldsymbol{\theta}\)와 무관한 상수. 따라서:
\[\operatorname*{arg\,min}_{\boldsymbol{\theta}}\;\mathcal{L}(\boldsymbol{\theta}) \;=\; \operatorname*{arg\,min}_{\boldsymbol{\theta}}\;\sum_{n=1}^{N}(y_n-\mathbf{x}_n^\top\boldsymbol{\theta})^2\]
결론: Gaussian 우도 하의 MLE = 최소자승 문제와 수학적으로 동일.

Q11MAP 추정에서 파라미터에 Gaussian prior \(p(\boldsymbol\theta)=\mathcal{N}(\mathbf{0},\boldsymbol\Sigma)\)를 사용하면 왜 L2 정규화와 동일한 결과가 나오는지 유도하라.유도+
풀이
Gaussian prior의 log (\(\boldsymbol{\Sigma}=\frac{1}{\lambda}\mathbf{I}\)):
\[\log p(\boldsymbol{\theta}) = -\frac{\lambda}{2}\left\|\boldsymbol{\theta}\right\|^2 + \text{const}\]
Gaussian 우도(→ 최소자승)와 결합:
\[\boldsymbol{\theta}_{\text{MAP}} = \operatorname*{arg\,min}_{\boldsymbol{\theta}}\;\frac{1}{N}\left\|\mathbf{y}-\mathbf{X}\boldsymbol{\theta}\right\|^2 + \lambda\left\|\boldsymbol{\theta}\right\|^2\]
∴ Gaussian prior MAP ≡ L2 정규화(Ridge Regression)
Q12잠재 변인 모델에서 잠재 변인 \(\mathbf{z}\)의 역할과, 우도 \(p(\mathbf{x}\mid\boldsymbol\theta)\)를 얻기 위해 \(\mathbf{z}\)를 어떻게 처리하는지 설명하라. 켤레 사전 분포가 왜 중요한가?개념+
풀이
잠재 변인 \(\mathbf{z}\)의 역할: 관측되지 않는 숨겨진 변인. 모델의 표현력을 높이면서 파라미터 수는 줄임. 예: GMM에서 각 데이터가 어떤 성분에서 나왔는지(z)는 관측 불가.
\[p(\mathbf{x}\mid\boldsymbol{\theta}) = \int p(\mathbf{x}\mid\mathbf{z},\boldsymbol{\theta})\;p(\mathbf{z})\;d\mathbf{z}\]
켤레 사전 분포의 중요성: prior \(p(\mathbf{z})\)가 likelihood의 켤레(conjugate)이면, posterior \(p(\mathbf{z}\mid\mathbf{x},\boldsymbol{\theta})\)가 같은 분포족 → 적분을 closed-form으로 계산 가능. 켤레가 아니면 MCMC, Variational Inference 등 수치 근사가 필요.
Q13Nested Cross-Validation(결합 교차검증)의 구조를 설명하고, inner loop와 outer loop의 역할을 명확히 구분하라.비교+
풀이
목적: 모델 선택(hyperparameter tuning)과 최종 성능 평가를 동시에 수행.

Inner loop: 각 후보 모델에 대해 K-fold CV 수행.
\[\mathbb{E}_{\mathcal{V}}\!\left[\mathbf{R}(\mathcal{V}\mid M)\right] \approx \frac{1}{K}\sum_{k=1}^{K} \mathbf{R}\!\left(\mathcal{V}^{(k)}\mid M\right)\]
→ 가장 낮은 error의 모델 선택.

Outer loop: Inner에서 선정된 최적 모델을 전체 Training 데이터로 재학습 → 별도 Test 데이터로 최종 generalization error 평가.
Q14두 모델 \(\mathcal{M}_1,\mathcal{M}_2\)에 대해 \(p(\mathcal{M}_1)=0.2\), \(p(\mathcal{M}_2)=0.4\), \(p(\mathcal{D}|\mathcal{M}_1)=0.3\), \(p(\mathcal{D}|\mathcal{M}_2)=0.15\)일 때, Bayes Factor와 Posterior Odds를 계산하고 결론을 내리라.계산+
풀이
\[\text{Bayes Factor} = \frac{p(\mathcal{D}\mid\mathcal{M}_1)}{p(\mathcal{D}\mid\mathcal{M}_2)} = \frac{0.3}{0.15} = 2.0\]
\[\text{Prior Odds} = \frac{p(\mathcal{M}_1)}{p(\mathcal{M}_2)} = \frac{0.2}{0.4} = 0.5\]
\[\text{Posterior Odds} = 0.5 \times 2.0 = 1.0\]
BF=2>1이므로 데이터 설명 측면에서 \(\mathcal{M}_1\)이 유리하지만, Prior Odds를 반영하면 Posterior Odds=1.0으로 두 모델이 동등하다. 추가 데이터 수집이 권장된다.
Q15베이지안 추론에서 predictive distribution \(p(\mathbf{x}_{\text{new}}\mid\mathcal{X})\)를 구하는 과정을 서술하고, MLE/MAP 방식과 근본적으로 다른 점을 설명하라.유도+
풀이
Bayesian Predictive Distribution:
\[p(\mathbf{x}_{\text{new}}\mid\mathcal{X}) = \int p(\mathbf{x}_{\text{new}}\mid\boldsymbol{\theta})\;p(\boldsymbol{\theta}\mid\mathcal{X})\;d\boldsymbol{\theta}\]
posterior \(p(\boldsymbol{\theta}\mid\mathcal{X})\)를 가중치로 하여 가능한 모든 \(\boldsymbol{\theta}\)에 대해 예측 분포를 평균화.

MLE/MAP와의 근본적 차이:
MLE/MAP: \(\boldsymbol{\theta}^*\) 하나를 고정 → \(p(\mathbf{x}_{\text{new}}\mid\boldsymbol{\theta}^*)\) 사용. 파라미터 추정 불확실성 무시.
Bayesian: 모든 가능한 \(\boldsymbol{\theta}\)를 확률적으로 가중평균 → 파라미터 불확실성을 예측에 반영.
Q16오캄의 면도날 원칙이 베이지안 모델 선택에서 어떻게 자동으로 구현되는지, 주변 우도(marginal likelihood)와 모델 복잡도의 관계를 통해 설명하라.비교+
풀이
\[p(\mathcal{D}\mid\mathcal{M}_k) = \int p(\mathcal{D}\mid\boldsymbol{\theta}_k)\;p(\boldsymbol{\theta}_k\mid\mathcal{M}_k)\;d\boldsymbol{\theta}_k\]
복잡한 모델: 파라미터 공간이 매우 넓음 → \(p(\boldsymbol{\theta}\mid M)\)이 넓게 퍼짐 → 특정 \(\boldsymbol{\theta}\)에서 밀도 낮음 → marginal likelihood 작아짐 (확률의 희석).

단순한 모델: 파라미터 공간이 좁음 → prior가 집중됨 → 해당 데이터를 설명하는 \(\boldsymbol{\theta}\)에서 밀도 높음 → marginal likelihood 클 수 있음.

결론: 주변 우도는 데이터 적합도와 모델 복잡도 사이의 자동 trade-off를 반영 → 오캄의 면도날의 베이지안 구현체.

d-분리 (d-Separation) 핵심 규칙
Blocked Paths in Bayesian Networks

방향 그래프에서 두 노드 집합이 조건부 독립인지 판별하려면 모든 경로가 차단(blocked)되는지 확인한다.

패턴
구조
중간 노드 관측 시
Chain
\(A \to C \to B\)
경로 차단 → A⊥B|C ✓
Fork
\(A \leftarrow C \rightarrow B\)
경로 차단 → A⊥B|C ✓
Collider ⚡
\(A \to C \leftarrow B\)
경로 활성화! → 독립 깨짐
⚡ Collider 주의
일반 패턴(Chain, Fork)과 반대로 동작. 관측하지 않으면 자동 차단(독립), 관측하면 오히려 경로가 열려 독립성이 깨진다. 후손(descendant)을 관측해도 마찬가지.
Q1
조건부 독립 판별 — T/F
d-Separation · 8점 (정답 +2, 오답 −1)

아래 확률분포의 방향그래프 모델을 보고, 변수들 간의 조건부 독립에 대한 서술의 참/거짓(T/F) 여부를 답하시오.

Q1 세부 문항
Q1-1 \(x \perp\!\!\!\perp y \mid a, d\) — 참(T)인가, 거짓(F)인가? 기출 +
풀이
답: F (독립 아님)

경로 \(x \to y\)가 직접 존재하며, a를 관측해도 이 직접 경로는 차단되지 않는다.
또한 d는 \(a \to d\) 와 \(y \to d\)의 collider인데, 이를 관측하면 오히려 \(a\)–\(y\) 경로가 활성화된다. 따라서 \(x \to a \to d \leftarrow y\) 경로도 열려 독립이 깨진다.
Q1-2 \(d \perp\!\!\!\perp z \mid x\) — 참(T)인가, 거짓(F)인가? 기출 +
풀이
답: F (독립 아님)

경로 \(d \leftarrow y \to z\): 여기서 y는 fork(tail-to-tail)이다.
x를 관측해도 y는 관측되지 않으므로, \(y\) 를 통한 경로는 열린 상태 (active).
→ d와 z는 x가 주어졌을 때도 독립이 아니다.
Q1-3 \(a \perp\!\!\!\perp b \mid c, d\) — 참(T)인가, 거짓(F)인가? 기출 +
풀이
답: T (독립 ✓)

a와 b 사이 경로: \(a \to d \leftarrow b\). 여기서 d는 collider.
d를 관측하면 경로가 열리지만, 이어지는 경로 \(d \to c\)에서 c도 관측되므로 chain이 차단된다.
다른 경로는 존재하지 않으므로 → \(a \perp\!\!\!\perp b \mid c, d\) 성립.
Q1-4 \(y \perp\!\!\!\perp c \mid x, b\) — 참(T)인가, 거짓(F)인가? 기출 +
풀이
답: F (독립 아님)

경로 \(y \to d \to c\): d는 chain(tail-to-head)이며 관측되지 않았다.
x와 b를 관측해도 d가 관측되지 않으므로 이 경로는 열려 있다.
→ y와 c는 x, b가 주어졌을 때도 독립이 아니다.
Q2
베이즈 요인 계산 및 해석
Bayes' Factor · 8점
문제
데이터 셋 \(\mathcal{D}\)에 대해 \(p(M_1)=0.25\), \(p(M_2)=0.75\)이고, 후위 이상치(Posteriori Odd)를 계산한 결과 \(\dfrac{p(\mathcal{D}|M_1)}{p(\mathcal{D}|M_2)} = 0.4\)를 얻었다.

(1) 베이즈 요인 \(B\)를 구하시오. (4점)
(2) 어느 모델이 데이터를 더 잘 설명하는가? (3점, 오답 −1점)
Q2-1 베이즈 요인(Bayes' Factor) \(B\)를 구하시오. 기출 +
풀이
핵심 공식:
\[\underbrace{\frac{p(\mathcal{D}|M_1)}{p(\mathcal{D}|M_2)}}_{\text{Posterior Odds}} = \underbrace{\frac{p(M_1)}{p(M_2)}}_{\text{Prior Odds}} \times \underbrace{B}_{\text{Bayes' Factor}}\]
대입:
\[0.4 = \frac{0.25}{0.75} \times B = \frac{1}{3} \times B \quad \therefore B = 1.2\]
Q2-2 베이즈 요인 \(B=1.2\)에 근거하여, 어느 모델이 데이터를 더 잘 설명하는가? 기출 +
풀이
\(B = 1.2 > 1\)이므로 \(M_1\)이 데이터를 더 잘 설명한다.

베이즈 요인 해석 기준 (Jeffreys scale):
  • \(B < 1\): \(M_2\) 지지
  • \(1 \le B < 3\): \(M_1\) 약하게 지지 (근거 불충분)
  • \(3 \le B < 10\): \(M_1\) 어느 정도 지지
  • \(B \ge 10\): \(M_1\) 강하게 지지
Q3
선형 회귀 MLE — 최대 우도 추정
Linear Regression · MLE · 15점
문제
로봇 청소기 동작 시간 \(x\)(분)와 청소 면적 \(y\)(m²) 데이터. 관계를 \(y=ax\)로 회귀분석하고자 한다.
x 3 7 15
y 5 14 31
(1) \(\mathcal{X}, \mathcal{Y}, \mathcal{D}\)를 집합 꼴로 작성하시오. (각 1점)
(2) 우도 함수 \(p(\mathcal{Y}|\mathcal{X},\boldsymbol{\theta})\) 유도 과정의 빈 칸을 채우시오. (각 1점)
(3) Design matrix \(\mathbf{X}\)를 구하고 \(\boldsymbol{\theta}_{ML} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}\)로 \(a\)를 구하시오. (7점)
Q3-1 입력 값 벡터 \(\mathcal{X}\), 결과 값 벡터 \(\mathcal{Y}\), Training set \(\mathcal{D}\)를 집합 꼴로 쓰시오. 기출 +
풀이
\[\mathcal{X} = \{3,\,7,\,15\}, \quad \mathcal{Y} = \{5,\,14,\,31\}\]
\[\mathcal{D} = \bigl\{\{3,5\},\;\{7,14\},\;\{15,31\}\bigr\}\]
Q3-2 Noise를 분산 \(\sigma^2\)인 정규 분포로 가정할 때, 우도 함수 \(p(\mathcal{Y}|\mathcal{X},\boldsymbol{\theta})\)를 유도하시오. 기출 +
풀이
i.i.d. 가정으로 분해:
\[p(\mathcal{Y}|\mathcal{X},\boldsymbol{\theta}) = p(y_1,y_2,y_3|x_1,x_2,x_3,\boldsymbol{\theta}) = \prod_{n=1}^{3} p(y_n\mid[\,x_n\,],[\,a\,])\]
모델 \(y = ax + \epsilon\), \(\epsilon \sim \mathcal{N}(0,\sigma^2)\) → \(y_n \sim \mathcal{N}(ax_n, \sigma^2)\):
\[= \prod_{n=1}^{3} \mathcal{N}\!\left(y_n\mid[\,ax_n\,],\,\sigma^2\right)\]
빈칸 정답: ① \(x_n\) (또는 \(\boldsymbol{\theta}\))  ② \(ax_n\)
Q3-3 Design matrix \(\mathbf{X}\)를 구하고, \(\boldsymbol{\theta}_{ML} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}\)로 \(a\)를 계산하시오. 기출 +
풀이
\(y = ax\) 모델 (절편 없음) → Design matrix는 \(x\) 열벡터 하나:
\[\mathbf{X} = \begin{bmatrix}3\\7\\15\end{bmatrix}, \quad \mathbf{y} = \begin{bmatrix}5\\14\\31\end{bmatrix}\]
\[\boldsymbol{\theta}_{ML} = a = \left(\mathbf{X}^T\mathbf{X}\right)^{-1}\mathbf{X}^T\mathbf{y} = \left([3\ 7\ 15]\begin{bmatrix}3\\7\\15\end{bmatrix}\right)^{-1}[3\ 7\ 15]\begin{bmatrix}5\\14\\31\end{bmatrix}\]
\[= \frac{1}{9+49+225}\times(15+98+465) = \frac{578}{283} \approx \boxed{2.042}\]

추가 연습 문제
Extra Practice · 관련 주제 심화
E1 Chain / Fork / Collider 패턴 판별: \(A \to C \to B\) (C 관측), \(A \leftarrow C \rightarrow B\) (C 미관측), \(A \to C \leftarrow B\) (C 미관측), \(A \to C \leftarrow B\) (C 관측) 각각 \(A \perp\!\!\!\perp B\) 성립 여부를 답하시오. 연습 +
풀이
  • (1) Chain, C 관측: 경로 차단 → \(A \perp\!\!\!\perp B \mid C\) ✓ (T)
  • (2) Fork, C 미관측: 경로 열림 → \(A \not\perp\!\!\!\perp B\) ✗ (F)
  • (3) Collider, C 미관측: 자동 차단 → \(A \perp\!\!\!\perp B\) ✓ (T) ← collider의 핵심!
  • (4) Collider, C 관측: 경로 활성화 → \(A \not\perp\!\!\!\perp B \mid C\) ✗ (F)
E2 어떤 DAG에서 \(p(A,B,C,D) = p(A)\,p(B)\,p(C|A,B)\,p(D|C)\)가 주어질 때, (1) \(A \perp\!\!\!\perp D \mid C\), (2) \(A \perp\!\!\!\perp B\), (3) \(A \perp\!\!\!\perp B \mid C\) 각각 성립 여부를 판별하시오. 연습 +
풀이
그래프: \(A \to C \leftarrow B\), \(C \to D\)
\[\text{(1) } A \perp\!\!\!\perp D \mid C \;\text{: T} \quad \text{(chain } A \to C \to D \text{에서 } C \text{ 관측 → 차단)}\]
\[\text{(2) } A \perp\!\!\!\perp B \;\text{: T} \quad (p(A,B)=p(A)\,p(B) \text{ 이므로 주변 독립})\]
\[\text{(3) } A \perp\!\!\!\perp B \mid C \;\text{: F} \quad \text{(}C\text{는 collider, 관측 시 경로 활성화!)}\]
E3 \(p(M_1)=0.6\), \(p(M_2)=0.4\), \(p(M_1|\mathcal{D})=0.7\), \(p(M_2|\mathcal{D})=0.3\)일 때, Posterior Odds, Prior Odds, Bayes' Factor를 각각 구하시오. 연습 +
풀이
\[\text{Posterior Odds} = \frac{0.7}{0.3} = \frac{7}{3}\]
\[\text{Prior Odds} = \frac{0.6}{0.4} = \frac{3}{2}\]
\[B = \frac{\text{Posterior Odds}}{\text{Prior Odds}} = \frac{7/3}{3/2} = \frac{14}{9} \approx 1.56\]
\(B > 1\)이므로 \(M_1\)이 데이터를 더 잘 설명한다.
E4 \(y = ax + b\) 모델로 데이터 \(\{(1,2),(2,4),(3,5)\}\)를 MLE로 추정하시오. Design matrix \(\mathbf{X}\)를 작성하고 Normal equation으로 \(a, b\)를 구하시오. 연습 +
풀이
절편 포함 모델 → Design matrix에 1 열 추가:
\[\mathbf{X} = \begin{bmatrix}1&1\\2&1\\3&1\end{bmatrix}, \quad \mathbf{y} = \begin{bmatrix}2\\4\\5\end{bmatrix}, \quad \boldsymbol{\theta}=\begin{bmatrix}a\\b\end{bmatrix}\]
\[\mathbf{X}^T\mathbf{X} = \begin{bmatrix}14&6\\6&3\end{bmatrix}, \quad \mathbf{X}^T\mathbf{y} = \begin{bmatrix}25\\11\end{bmatrix}\]
\[(\mathbf{X}^T\mathbf{X})^{-1} = \frac{1}{6}\begin{bmatrix}3&-6\\-6&14\end{bmatrix}\]
\[\boldsymbol{\theta}_{ML} = \frac{1}{6}\begin{bmatrix}3&-6\\-6&14\end{bmatrix}\begin{bmatrix}25\\11\end{bmatrix} = \frac{1}{6}\begin{bmatrix}9\\4\end{bmatrix} \;\Rightarrow\; a=1.5,\; b=\tfrac{2}{3}\]
E5 가우시안 노이즈 가정 하 선형 회귀의 Log-likelihood를 구하고, 이를 최대화하는 것이 최소제곱법(Least Squares)과 동치임을 보이시오. 연습 +
풀이
단일 관측의 우도: \(p(y_n|x_n,\mathbf{w}) = \mathcal{N}(y_n \mid \mathbf{w}^T\boldsymbol{\phi}(x_n), \sigma^2)\)
\[\ln p(\mathbf{y}|\mathbf{X},\mathbf{w}) = -\frac{N}{2}\ln(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_{n=1}^{N}\!\left(y_n - \mathbf{w}^T\boldsymbol{\phi}(x_n)\right)^2\]
Log-likelihood 최대화 ≡ \(\displaystyle\sum_{n=1}^N (y_n - \mathbf{w}^T\boldsymbol{\phi}(x_n))^2\) 최소화 = 최소제곱법(OLS).
즉, 가우시안 노이즈 가정 하에서 MLE = OLS는 수학적으로 완전히 동일하다.
E6 MLE의 단점과 이를 극복하기 위한 MAP, 베이지안 추론의 차이점을 비교하여 설명하시오. 연습 +
풀이
MLE의 단점: 파라미터를 하나의 점 추정값으로 고정 → 파라미터 불확실성 무시, 소량 데이터에서 overfitting 위험.
\[\boldsymbol{\theta}_{ML} = \underset{\boldsymbol{\theta}}{\arg\max}\; p(\mathcal{D}|\boldsymbol{\theta})\]
MAP: prior \(p(\boldsymbol{\theta})\) 추가 → 정규화 효과, 여전히 점 추정.
\[\boldsymbol{\theta}_{MAP} = \underset{\boldsymbol{\theta}}{\arg\max}\; p(\boldsymbol{\theta}|\mathcal{D}) = \underset{\boldsymbol{\theta}}{\arg\max}\; p(\mathcal{D}|\boldsymbol{\theta})\,p(\boldsymbol{\theta})\]
Bayesian: posterior 전체를 유지 → 파라미터 불확실성 정량화, 계산 비용 큼.
\[p(\mathbf{x}_{\text{new}}|\mathcal{D}) = \int p(\mathbf{x}_{\text{new}}|\boldsymbol{\theta})\,p(\boldsymbol{\theta}|\mathcal{D})\,d\boldsymbol{\theta}\]
